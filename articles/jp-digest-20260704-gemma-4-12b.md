「またローカルLLM（自分のPC上で動かす大規模言語モデル）の新しいやつが出たらしいよ」——そう聞いても、正直もうお腹いっぱい…って人、多いですよね。わかります。でも今回の **Gemma 4 12B** はちょっと立ち止まる価値があるんです。なにせ **メモリ16GBのノートPC1台で、画像も音声もいける軽量モデル**が動いちゃう。クラウドもGPUサーバーもいらない。

この記事は、海外の公式発表・開発者向けガイド・実行者の解説など**複数の記事を突き合わせて、自分の言葉で再構成**したものです。「で、結局なにがすごくて、どう動かすの？」を、数字とコマンド込みで一気に整理していきますね。

![16GBのノートPCで動く軽量マルチモーダルLLM Gemma 4 12B](/images/jp-digest-20260704-gemma-4-12b/figure1.png)

## そもそも何が起きたの？

まず時系列から。Gemma 4ファミリー自体は2026年4月2日に登場して、そこから **2026年6月3日に「12B（120億パラメータ）」版**が追加された、という流れです。パラメータ数っていうのは、ざっくり言えばモデルの「脳みその大きさ」。数字が大きいほど賢くなりやすいけど、その分メモリも食う、というトレードオフがあるんですね。

で、この12Bのポジションが絶妙で。スマホ向けの小さいやつ（E2B/E4B）と、ワークステーション向けの大きいやつ（26B・31B）のちょうど真ん中。**「そこそこ賢いのに、普通のPCで動く」**という一番おいしいゾーンを狙ってきたわけです。

海外のコーディング系まとめでも、2026年半ばの「手元で動かせるモデル」として名前がよく挙がるようになりました。

![出典元メディアの見出し](/images/jp-digest-20260704-gemma-4-12b/source1.png)
*出典: DEV Community（https://dev.to/zyvop/the-best-open-source-llms-for-coding-right-now-june-2026-n10）*

## いちばんの変化は「画像も音声も、同じ頭で読む」

技術的にいちばん面白いのがここ。Gemma 4は **エンコーダーフリー（encoder-free）の統一アーキテクチャ**を採用しています。

……と言われても「?」ですよね。かみ砕きます。これまでのマルチモーダル（画像や音声も扱える）モデルって、画像を理解するための専用の変換器（エンコーダ）、音声用の変換器…と、入力の種類ごとに別々の部品を挟んでから言語モデルに渡すのが普通だったんです。

Gemma 4はその橋渡し部品を減らして、**画像も音声も文章も、言語のバックボーンに直接ぶち込む**設計にした。しかも、中規模のGemmaで**初めて音声入力にネイティブ対応**したモデルでもあります。部品が減れば軽くなるし速くなる。理にかなってるんですよね。

![エンコーダーフリーの統一アーキテクチャ：画像も音声も同じ頭で読む](/images/jp-digest-20260704-gemma-4-12b/figure2.png)

実際に扱えるのはテキスト・画像・音声・動画。ある解説では、5分ほどの動画を毎秒1フレームで読ませる、みたいな使い方も紹介されていました。対応言語も140以上と幅広くて、日本語ももちろん入っています。

## 前世代を「サイズ差」で逆転した数字

「軽いのはわかった。でも軽い＝バカなんでしょ？」——いい疑問です。ここが今回のキモ。

複雑な推論を測る **BBEH** というベンチマーク（性能テスト）で、Gemma 4 12Bは **53**。前世代の Gemma 3 27B が **約18** だったので、大幅な伸びなんです。しかも相手は**パラメータが2倍以上ある上位モデル**。それを、半分以下のサイズで上回った、と。サイズを積むだけじゃない進化がちゃんとあるわけです。

![2倍サイズの前世代をBBEHで逆転](/images/jp-digest-20260704-gemma-4-12b/figure3.png)

他の指標でも、12B版はGPQA Diamond（大学院レベルの難問）で78.8、LiveCodeBench（コード生成）で72あたりと高水準。

ひとつ正直に言っておくと、**ベンチマークの数値は媒体やモデルサイズによって結構バラつきます**。26Bや31Bの数字と混ざって紹介されていることもあるので、「12Bでこの数字」と「上位サイズでこの数字」は分けて読むのが安全です。この記事では12Bにひもづく数字を中心に拾っています。

## 手元のPCでどこまで動く？メモリの話

ここが一番実用的なところ。12Bを動かすのに必要なメモリの目安は、量子化（モデルを圧縮して軽くする技術）のかけ方でこう変わります。

| 精度 | 必要メモリの目安 |
|------|------------------|
| 4bit量子化（いちばん軽い） | 7〜8GB |
| 8bit（画質と軽さの中間） | 13〜14GB |
| BF16（フル精度） | 25GB |

![Gemma 4 12Bを動かすのに必要なメモリ目安](/images/jp-digest-20260704-gemma-4-12b/figure4.png)

つまり、**メモリ16GBのノートPCなら4bitや8bitで普通に動く**んです。25GBのフル精度はさすがにワークステーション級が要りますが、日常づかいなら4bitで十分。ここが「クラウド課金なしで、手元で完結できる」という今回の売りにつながっています。

なお、統合メモリ（CPUとGPUでメモリを共有する仕組み）のノートPCでも動くよ、と公式ガイドでは案内されていました。

## 使い方：コマンド2つでローカル実行

お待たせしました、実際の動かし方です。いちばん手軽なのはモデル実行ツールを使う方法。**コマンド2つで起動**まで行けます。

```bash
# 1. 実行ツールを入れる（macOSの例）
brew install ollama

# 2. モデルを呼ぶだけ（初回はモデルを自動ダウンロード）
ollama run gemma4:12b
```

これだけ。あとはターミナルに日本語で話しかければOKです。サイズ違いを試したいときは `gemma4:e2b` `gemma4:e4b` `gemma4:26b` `gemma4:31b` のようにタグを変えるだけ。画像や音声を渡すマルチモーダルな使い方もできます。

![コマンド2つでローカル実行する手順](/images/jp-digest-20260704-gemma-4-12b/figure5.png)

生成のクセを整えたいなら、配布元が推奨している設定値がこちら。

- **temperature（ランダムさ）**: 1.0
- **top-p**: 0.95
- **top-k**: 64

ファインチューニング（自分のデータで追加学習）までやりたい人向けには、専用のローカルUIツールや `llama.cpp` 経由の方法も用意されています。まずは `ollama run` で触ってみて、物足りなくなったら踏み込む、くらいの温度感でいいと思います。

## どのサイズを選ぶ？ラインナップ早見

12Bだけじゃなく、ファミリー全体の住み分けも見ておくと選びやすいです。

- **E2B / E4B**：スマホやエッジ（端末側）向けの超軽量。コンテキスト長（一度に読める情報量）は128Kトークン
- **12B**：今回の主役。ノートPCで動く万能タイプ。コンテキスト256Kトークン
- **26B（MoE）**：MoE（専門家混合：必要な部分だけ動かして効率化する仕組み）採用で、コンシューマー向けGPUで高品質
- **31B**：最高品質のフラッグシップ。ワークステーション向け

ライセンス面では、ある解説では**商用利用OK・利用者数の上限（MAU制限）もなし**と紹介されていました。ただしライセンスの細かい条件は変わりうるので、業務で使うなら**公式の利用規約を必ず自分で確認**してくださいね。ここは念のため。

## 注意しておきたいこと

いいことばかり書いてきたので、冷静な話も。

- **12Bはあくまで中規模**。込み入った長文推論や難しいコードでは、クラウドの巨大モデルにまだ及ばない場面はあります。適材適所で。
- **数字は鵜呑みにしない**。前述のとおりベンチマークは媒体差・サイズ差が大きいので、自分の用途で軽く試すのが結局いちばん確実です。
- **音声・動画対応は環境依存**。ツールやバージョンによって使える機能が違うので、マルチモーダルをフルに使いたいなら対応状況を確認しましょう。

## まとめ

Gemma 4 12Bのポイントを最後にぎゅっと。

- 2026年6月3日登場。**16GBのノートPCで動く軽量マルチモーダルモデル**
- エンコーダーフリー設計で、**画像・音声・文章を同じ頭で処理**（中規模Gemmaで初のネイティブ音声入力）
- BBEHで前世代の**2倍サイズのモデルを逆転**するなど、小さいのに賢い
- 使い方は `ollama run gemma4:12b` の**実質2コマンド**でローカル起動

「賢いAIはクラウドで課金して使うもの」という前提が、こういう軽量モデルでじわじわ崩れてきてるのが面白いところ。まずは手元のPCで `ollama run` してみて、返ってくる日本語の自然さを体感してみてください。想像より“ちゃんと”してて、ちょっと驚くはずです。
