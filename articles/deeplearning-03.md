この記事を読み終わるころには、「勾配降下法ってこういうことか」「学習率（がくしゅうりつ／learning rate）ってこう効くのか」が、山登り（というか山下り）のイメージでスッと腹落ちしているはずです。数式は最小限、たとえ話多め、そして Python のコードも動く形で用意しました。気軽についてきてくださいね。

![谷の底に向かってボールが転がり落ちていくイメージ。勾配降下法の基本のイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-03/figure1.png)

## そもそも勾配降下法で何がしたいの？

深層学習のモデルには、**重み（おもみ／weight）** と呼ばれる数字がたくさん詰まっています。学習とは、この重みを調整して「予測のズレ（＝誤差）」をできるだけ小さくしていく作業でした。

このズレの大きさを表す関数を **損失関数（そんしつかんすう／loss function）** と呼びます。イメージは「間違いの多さメーター」。損失が大きいほどモデルはポンコツ、小さいほど賢い、というわけです。

だから深層学習のゴールは、**「損失関数をできるだけ小さくする重みを見つける」** こと。これって、地形にたとえると **「谷のいちばん低いところ（=損失が最小の場所）を探して降りていく」** のと同じなんです。勾配降下法は、その「降り方」を教えてくれる方法なんですね。

では、どっちに降りればいいのか。その「向き」を教えてくれるのが、前回の微分をパワーアップさせた **偏微分** と **勾配** です。

## 偏微分 ── ひとつの変数だけに注目する微分

前回の微分は、入力がひとつ（$x$ だけ）の関数が相手でした。でも実際の損失関数は、重みが何百万個もある **多変数関数（たへんすうかんすう／複数の入力を持つ関数）** です。$f(x, y)$ のように入力が複数あるとき、「どっちを動かしたときの変化を見るの？」という問題が出てきます。

そこで登場するのが **偏微分（へんびぶん／partial derivative）** です。やることはシンプルで、**「注目する変数以外は“定数”だと思って固定し、ひとつの変数についてだけ微分する」** だけ。

![3次元の曲面の上で、x方向とy方向それぞれの傾きを1方向ずつ測るイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-03/figure2.png)

山の斜面に立っている自分を想像してください。「**東西方向にだけ**足を踏み出したら、標高はどれだけ変わる？」を測るのが $x$ についての偏微分。「**南北方向にだけ**動いたら？」を測るのが $y$ についての偏微分。ほかの向きのことはいったん忘れて、1方向ずつ調べる——それが偏微分の気持ちです。

記号では、丸っこい d（デル、$\partial$）を使って $\dfrac{\partial f}{\partial x}$ と書きます。「$f$ を $x$ について偏微分したもの」という意味です。読み方が難しいだけで、やってることは「$y$ を数字だと思って、$x$ でいつも通り微分」なだけ。身構えなくて大丈夫です。

## 勾配 ── いちばん急な向きを指す矢印

各変数の偏微分がそろったら、それを1本のベクトル（矢印）にまとめます。これが **勾配（こうばい／gradient）** です。記号は $\nabla f$（ナブラ・エフ）。

$$
\nabla f = \left( \frac{\partial f}{\partial x},\ \frac{\partial f}{\partial y} \right)
$$

この矢印には、とても大事な性質があります。それは——**勾配は「その地点でもっとも急に“上る”方向」を指している**、ということ。しかも矢印の長さは「その向きの傾きの急さ」を表します。

![地形を上から見た等高線上で、各点の矢印がいちばん急な上り方向を指しているイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-03/figure3.png)

つまり勾配は「ここから最速で山を登るなら、こっちだよ」と指し示す矢印。ということは……**その逆向き（マイナス方向）に進めば、最速で山を下れる** わけです。これが勾配降下法のいちばんのキモ。損失という山を下りたいなら、勾配の**逆向き**に進めばいい。とてもシンプルな発想ですよね。

## 勾配降下法 ── 逆向きにちょっとずつ下る

いよいよ本題です。勾配降下法のルールは、たった1本の更新式で書けます。重みを $w$、勾配を $\nabla f$、そして進む歩幅を $\eta$（イータ）とすると：

$$
w_{\text{新}} = w_{\text{旧}} - \eta \, \nabla f
$$

日本語に訳すとこうです。**「新しい重み ＝ 今の重み − (歩幅 × 勾配)」**。勾配の前にマイナスが付いているのがポイント。さっき言ったとおり、勾配は「上り」の向きなので、**マイナスを付けて逆向き（下り）にする** んですね。

この計算を1回やると、谷に向かって1歩だけ近づきます。あとはこれを何度も繰り返すだけ。少しずつ、少しずつ谷底へ。まるで霧の中で足元の傾きだけを頼りに、いちばん低いところを目指して下りていく感じです。

![谷の斜面を、点が少しずつステップしながら底に向かって下りていくイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-03/figure4.png)

### Pythonで実際に谷を下ってみる

言葉だけだとふわっとするので、実際に手を動かしましょう。いちばん簡単な谷、$f(x) = x^2$ を下ってみます。この関数の勾配（今回は変数が1つなので、ただの微分）は $f'(x) = 2x$ です。谷底は $x=0$。そこにちゃんと近づけるか、Python で確かめます。

```python
# 損失(谷)の関数と、その勾配(傾き)
def f(x):
    return x ** 2       # 下りたい谷。x=0 が谷底

def grad(x):
    return 2 * x        # f(x)=x^2 の微分(勾配)は 2x

x = 5.0                 # スタート地点(谷のちょっと上のほう)
lr = 0.1                # 学習率(歩幅) η。とりあえず 0.1

# 20回ぶん、谷を下ってみる
for step in range(20):
    g = grad(x)         # 今いる場所の勾配(=上り方向の傾き)を計算
    x = x - lr * g      # 勾配の「逆向き」に、歩幅ぶんだけ動く ← これが更新式
    print(f"step {step+1:2d}: x = {x:.4f}, 損失 = {f(x):.4f}")
```

実行すると、$x$ が 5.0 からだんだん 0 に近づき、損失も小さくなっていくのが見えます（最後は $x \approx 0.06$ くらい、損失もほぼ 0）。特別な魔法は使っていません。「勾配を計算して、その逆向きに歩幅ぶん動く」を繰り返しているだけ。これが勾配降下法の全部です。

## 学習率 ── 歩幅が大きすぎても小さすぎてもダメ

さっきの式に出てきた歩幅 $\eta$ が、**学習率（learning rate）** です。深層学習でいちばん有名で、いちばん悩ましいツマミがこれ。「1歩でどれだけ進むか」を決める数字です。

これは大きすぎても小さすぎてもうまくいきません。ちょうどいい塩梅が大事なんです。

![学習率が小さい・ちょうどいい・大きい、3パターンの下り方を比べたイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-03/figure5.png)

- **学習率が小さすぎる**：歩幅が小さすぎて、谷底になかなかたどり着けません。ちょっとずつしか進めないので、学習に時間がかかりすぎます。慎重すぎてなかなか前に進めない人、みたいなイメージ。
- **学習率がちょうどいい**：気持ちよく、まっすぐ谷底へ。理想はこれ。
- **学習率が大きすぎる**：これがいちばん危険。歩幅がデカすぎて、谷底を **飛び越えて反対側の壁** に着地してしまう。しかも次はもっと大きく跳ね返り……と、下るどころか **どんどん上っていってしまう（発散）** こともあります。

![学習率が大きすぎて谷底を飛び越え、振動しながら発散していくイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-03/figure6.png)

さっきのコードで `lr = 0.1` を `lr = 1.1` に変えてみると、この「発散」が体感できます。$x$ の値が 0 に近づくどころか、どんどん大きくなって数字が吹っ飛んでいくはず。損失が減らずに逆に増えていくときは、「学習率が大きすぎるのかも？」と真っ先に疑うのが定石です。

まずは小さめ（`0.01` や `0.001` あたり）から試して、様子を見ながら調整する——これが実務でもよくやる進め方です。

## 谷がひとつとは限らない ── 局所解の話

ここまでは「谷はきれいなお椀型」という前提で話してきました。$f(x)=x^2$ みたいに、谷がひとつだけならラクなんです。でも実際の深層学習の損失関数は、あちこちにボコボコと谷がある **でこぼこ地形** をしています。

![浅い谷がいくつもあり、その中にひとつだけ深い谷がある損失地形のイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-03/figure7.png)

すると困ったことが起きます。勾配降下法は「足元の傾き」しか見ていないので、**本当のいちばん低い谷（大域最適解／だいいきさいてきかい）** ではなく、**手前の浅い谷（局所最適解／きょくしょさいてきかい）** で「もう平らだ、着いた！」と満足して止まってしまうことがあるんです。ハイキングで、本当の谷底じゃなくて途中のくぼみで足を止めちゃう感じですね。

ただ、ここで悲観しなくて大丈夫。深層学習の世界では「多少浅い谷で止まっても、実用上は十分いい性能が出る」ことが多いと経験的に知られています。さらに、この局所解にハマりにくくする工夫（次回以降に少しずつ出てくる、確率的な更新や慣性をつける手法など）もちゃんと用意されています。今は「谷はひとつとは限らない」という感覚だけ持っておけばOKです。

## 勾配降下法の流れをまとめると

ここまでの流れを、ぐるっと1周のループに整理しておきましょう。深層学習の学習は、ざっくりこの繰り返しです。

![勾配を計算→重みを更新→繰り返し、という円環のループのイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-03/figure8.png)

1. **今の重みで損失（間違いの多さ）を計算する**
2. **勾配（＝損失を最も増やす向き）を計算する**
3. **勾配の逆向きに、学習率ぶんだけ重みを更新する**
4. **1に戻って繰り返す**（損失が十分小さくなるまで）

たったこれだけ。「勾配で向きを知り、学習率で歩幅を決め、逆向きにちょっと動く」を延々と繰り返す。これが、あの巨大なニューラルネットワークが賢くなっていくときにも、内部で起きていることの本質です。仕組みが分かると、なんだか親近感がわいてきませんか？

## まとめ

今回のポイントをおさらいします。

- **偏微分** は「注目する変数以外を固定して、ひとつの変数だけで微分する」こと。
- その偏微分を集めたベクトルが **勾配** で、**もっとも急な“上り”の向き** を指す。
- **勾配降下法** は、その **逆向き** に **学習率（歩幅）** ぶんずつ重みを更新し、損失という谷を少しずつ下っていく方法。式は「新しい重み ＝ 今の重み − 学習率 × 勾配」の1本だけ。
- **学習率** は大きすぎると発散、小さすぎると遅い。まずは小さめから試すのが安全。
- 実際の損失地形はでこぼこで、**局所解** で止まることもあるが、実用上はうまくいくことが多い。

「坂道を、いちばん急な向きの逆に、ちょっとずつ下る」——この一言さえ持ち帰ってもらえれば、今回は大成功です。
