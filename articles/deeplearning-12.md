## 順伝播（forward propagation）って何？

順伝播（じゅんでんぱ／forward propagation）とは、入力したデータが、入力層（にゅうりょくそう） → 隠れ層（かくれそう） → 出力層（しゅつりょくそう）と一方向に流れていって、最後に予測（答え）が出てくるまでの計算の流れのことです。

forward（前へ）に、propagation（伝わる）していくので、順伝播なんですね。データは後ろへ戻らず、まっすぐ前へ進みます。

たとえば、ベルトコンベアを思い浮かべてみてください。材料（入力データ）をベルトコンベアに乗せると、各工程（層）で少しずつ加工されて、最後に製品（予測）が出てきますよね。途中の工程を逆流するわけではなく、前へ前へ進んでいくイメージです。

今回やることは、この「順伝播＝予測を出す計算」をコードにすることです。次回は、その逆向きに誤差を後ろへ流す逆伝播（ぎゃくでんぱ／backpropagation）を学びます。まずは今日、前向きの流れをしっかり手で動かしてみましょう。

![入力層・隠れ層・出力層を、信号が前へ前へと流れていく](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-12/figure2.png)

## 層の計算は「行列積」でまとめてやる

前回まで、ニューロン（neuron／神経細胞のような計算単位）1個ずつが「入力×重みを足して、バイアスを足す」という計算をしていました。

でも、層にニューロンがたくさんあると、これを1個ずつ書くのは大変です。ニューロンが3個、10個、100個……と増えたら、手で全部書くのはつらいですよね。

そこで登場するのが、行列（ぎょうれつ／matrix）と行列積（ぎょうれつせき／matrix product、行列同士のかけ算）です。層全体の計算を、行列のかけ算1回でまとめて表せます。

1つの層の計算は、次のように書けます。

$$z = xW + b$$

ここで x は入力（ベクトル）、W は重み（行列／weight）、b はバイアス（かたより／bias）、z はその層の計算結果です。

そして、z を活性化関数 f に通して、その層の出力 a を作ります。

$$a = f(z)$$

ポイントは、1個ずつ掛け算するより、行列積でまとめた方が「速いし・コードも短いし・見た目もスッキリ」することです。NumPy（なむぱい／数値計算ライブラリ）は、この行列積がとても得意なんです。

![バラバラのかけ算を、1回の行列積にまとめるイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-12/figure3.png)

## まずはNumPyの準備運動

NumPy（なむぱい）は、Pythonで数値計算・行列計算をするための定番ライブラリです。コードでは `import numpy as np` と書いて、`np` という名前で使うのが慣習です。

配列（はいれつ／array）を作るには、`np.array([...])` と書きます。1次元ならベクトル、2次元なら行列を表せます。

行列積、つまり行列のかけ算は `np.dot(A, B)` または `A @ B` と書けます。今回は分かりやすさ重視で `np.dot` を使います。

もうひとつ、ブロードキャスト（broadcast／形の違う配列をうまくそろえる仕組み）も出てきます。今回は「行列 + バイアス（ベクトル）」が、この仕組みのおかげで自然に足せるんです。

まずは、NumPyの配列を少し触ってみましょう。

```python
import numpy as np

a = np.array([1, 2, 3])          # 1次元配列(ベクトル)
b = np.array([[1, 2], [3, 4]])   # 2次元配列(行列)

print(a.shape)   # (3,)   要素3つのベクトル
print(b.shape)   # (2, 2) 2行2列の行列
```

ここで出てくる `shape`（しぇいぷ／形）は、配列の「形」を表します。たとえば `(3,)` なら要素が3つのベクトル、`(2, 2)` なら2行2列の行列です。

順伝播では、この shape がとても大事になります。「数字の形が合っているか」を見るクセがつくと、エラーで迷子になりにくくなりますよ。

![数字が格子状に並んだNumPyの配列(行列)のイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-12/figure4.png)

## 1つの層をコードにしてみる

では、さっきの数式をそのままコードにしていきましょう。

層の計算は、

$$z = xW + b$$

そして、

$$a = f(z)$$

でしたね。

ここではまず、活性化関数であるシグモイドとReLUをNumPyで書いておきます。

```python
def sigmoid(x):
    return 1 / (1 + np.exp(-x))

def relu(x):
    return np.maximum(0, x)
```

`np.exp` は指数関数（しすうかんすう／eの累乗）です。シグモイドは、前回学んだ `1/(1+e^-x)` の式を、そのままPythonで書いています。

ReLUは `np.maximum(0, x)` で書けます。これは「0より小さければ0、大きければそのまま」を一発で表しています。

前回は数式として見ていたものが、そのままコードになるんです。ここ、ちょっと感動ポイントですよね。

![掛けて足す→活性化関数を通す、という1つの層の処理の流れ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-12/figure5.png)

## 活性化関数をもう一度おさらい

ここで、活性化関数をもう一度だけおさらいしておきましょう。

シグモイド（sigmoid）は、どんな数字も0〜1の間に押し込む、なめらかなS字カーブです。数式は次の通りです。

$$\sigma(x) = \frac{1}{1 + e^{-x}}$$

ReLU（れる／Rectified Linear Unit）は、マイナスは0に、プラスはそのまま通す折れ線です。

$$\mathrm{ReLU}(x) = \max(0, x)$$

どちらも、ニューラルネットワークに非線形性（ひせんけいせい）を生むために、層と層の間に入れます。もし活性化関数がないと、層を何枚重ねても表現力が増えにくい、という話を前回しましたね。

今回のコードでは、隠れ層にシグモイドを使います。

![なめらかなS字のシグモイドと、折れ線のReLU](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-12/figure6.png)

## 2層ニューラルネットワークの順伝播を完成させる

いよいよ本番です。

今回は、入力2個 → 隠れ層3個 → 出力1個の、小さな2層ニューラルネットワークの順伝播を書きます。

重み W とバイアス b は、本来は学習で決める値です。ただし今回は「順伝播の流れ」を体感するために、手で適当な値を入れておきます。つまり、まだ学習はしていません。

では、コードを見てみましょう。

```python
import numpy as np

def sigmoid(x):
    return 1 / (1 + np.exp(-x))

# 入力(2次元のデータ1件)
x = np.array([1.0, 0.5])

# 第1層(隠れ層): 入力2 -> 隠れ3
W1 = np.array([[0.1, 0.3, 0.5],
               [0.2, 0.4, 0.6]])
b1 = np.array([0.1, 0.2, 0.3])

# 第2層(出力層): 隠れ3 -> 出力1
W2 = np.array([[0.1],
               [0.2],
               [0.3]])
b2 = np.array([0.4])

# --- 順伝播 ---
z1 = np.dot(x, W1) + b1   # 隠れ層の入力
a1 = sigmoid(z1)          # 隠れ層の出力(活性化)
z2 = np.dot(a1, W2) + b2  # 出力層の入力
y = z2                    # 今回は出力をそのまま予測とする

print("z1 =", z1)
print("a1 =", a1)
print("y  =", y)
```

上から順に、ていねいに見ていきましょう。

1つ目、`x` が入力データです。今回は `[1.0, 0.5]` という2個の数字を持つデータ1件です。

2つ目、`W1` と `b1` が1層目、つまり隠れ層の重みとバイアスです。`np.dot(x, W1)` で行列積を計算し、そこに `+ b1` でバイアスを足して、`z1` になります。

3つ目、`z1` をシグモイドに通して `a1` を作ります。`a1` は隠れ層の出力です。ここで数字が0〜1の範囲に変換されます。

4つ目、`a1` に2層目の `W2` と `b2` を使って同じように計算します。これが `z2` です。今回は、この `z2` をそのまま予測 `y` とします。

つまり、`x` が前へ前へ流れて、最後に `y` になりました。これがまさに順伝播です。

実行結果は、このとおりです。

- z1 = [0.3 0.7 1.1]
- a1 = [0.57444252 0.66818777 0.75026011]
- y  = [0.81615984]

小数点以下がずらっと並んでいますが、これはPythonがそのまま細かく表示しているだけです。ざっくり `a1` は `[0.57, 0.67, 0.75]`、`y` は `0.82` くらい、と読めばOKですよ。`a1` はシグモイドを通したので、ちゃんと0〜1の値になっていますね。

これで、ニューラルネットワークが1つの予測（およそ `0.82`）を出せました！

ただし、この値の「良し悪し」はまだ気にしなくてOKです。なぜなら、重みはまだ適当に決めただけで、学習していないからです。今は「入力が流れて、予測が出る」という順伝播の形が見えれば大成功です。

![2つの層を信号が流れて、最後に予測が出てくる](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-12/figure7.png)

## データをまとめて流す「バッチ」

実際の学習では、データを1件ずつではなく、何件もまとめて流すことがよくあります。これをバッチ（batch／ひとかたまり）と呼びます。

うれしいのは、行列積で書いておくと、コードをほとんど変えずに複数データを一気に処理できることです。入力を2次元にするだけで、同じ考え方で動きます。

さっそく、2件の入力データをまとめて流してみましょう。

```python
# 入力を「2件まとめて」にする(2行 = 2件のデータ)
X = np.array([[1.0, 0.5],
              [0.2, 0.9]])

Z1 = np.dot(X, W1) + b1
A1 = sigmoid(Z1)
Y  = np.dot(A1, W2) + b2

print("X.shape =", X.shape)  # (2, 2)
print("Y.shape =", Y.shape)  # (2, 1)
print("Y =", Y)
```

実行結果は、このとおりです。

- X.shape = (2, 2)
- Y.shape = (2, 1)
- Y = [[0.81615984]
   [0.80321786]]

`X.shape = (2, 2)` は、「2件のデータがあり、それぞれ入力が2個」という意味です。そして `Y.shape = (2, 1)` は、「2件それぞれに、出力が1個ずつある」という意味です。

1件目の結果は約 `0.82`（`0.81615984`）で、さっき1件だけ流したときと同じですね。2件目は約 `0.80`（`0.80321786`）になりました。

行列積で書いたおかげで、1件でも複数件でも同じようなコードで動きます。これが、行列で計算する大きなメリットなんです。

![複数のデータをまとめて一気に流すバッチのイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-12/figure8.png)

## つまずきやすい「shape(形)」の話

順伝播で初心者が一番ハマりやすいのが、shape（しぇいぷ／形）の不一致エラーです。

行列積 `np.dot(A, B)` は、Aの列数とBの行数が一致していないと計算できません。ここはとても大事です。

今回の例で見ると、入力は2個でした。だから `W1` は「2行 × 3列」でないといけません。最初の「2」が入力の数に合っています。そして「3列」なので、隠れ層のニューロンは3個になります。

次の `W2` は「3行 × 1列」です。前の層の出力が3個なので、次の重みの行数も3にそろえる必要があります。

言葉でつなげると、こうです。

入力2 →（W1: 2×3）→ 隠れ3 →（W2: 3×1）→ 出力1

数字が数珠つなぎでつながっているのが分かればOKです。

もしエラーが出たら、まず `print(A.shape)` のようにして形を確認してみてください。「どこで形が合っていないかな？」と見るクセをつけると、順伝播のコードはかなり読みやすくなりますよ。

## まとめ

今回は、これまで絵で見てきたニューラルネットワークを、Python + NumPyで実際に動かしました。

順伝播とは、データが入力から出力へ前へ流れて、予測を出す計算のことでしたね。

1つの層は、

$$z = xW + b$$

そして、

$$a = f(z)$$

で表せます。これを行列積でまとめて書くと、短く・速く・バッチ対応までできるようになります。

今回のポイントをまとめると、こんな感じです。

- 順伝播は、入力層 → 隠れ層 → 出力層へ前向きに進む計算
- 1つの層は `z = xW + b`、そのあと活性化関数で `a = f(z)` にする
- NumPyの `np.dot` を使うと、層全体の計算を行列積でまとめて書ける
- 小さな2層ネットワークを動かして、予測 `y ≒ 0.82`（`0.81615984`）を出せた
- 複数データをまとめて流すバッチでも、行列積なら自然に処理できる

ただし、今の重みは適当な値のままです。「正しい予測」に近づけるには、重みを調整する必要があります。つまり、学習が必要なんですね。
