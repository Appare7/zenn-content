## そもそも「機械学習」とは？

機械学習をひとことで言うと、こうです。

> **データからパターン（規則性）を自動で見つけ出し、まだ見ていないデータに対して予測する仕組み**

……と言われてもピンと来ないと思うので、たとえ話でいきましょう。

あなたが小さい子どもに「これは犬だよ」「これは猫だよ」と、写真を何百枚も見せていくとします。最初はめちゃくちゃな答えを言うけれど、たくさん見せているうちに、子どもは自分で「耳がとがってて、ヒゲがあって……これは猫っぽい」というコツを **自分でつかんでいきます**。そして、一度も見たことのない新しい写真を見せても「これは猫！」と当てられるようになる。

機械学習がやっているのは、まさにこれです。人間が「猫とは、耳がこうで、目がこうで……」と **ルールを1つ1つ書いてあげる代わりに**、大量のデータを見せて **コンピュータ自身にコツ（パターン）をつかませる**。ここが最大のポイントです。

## 昔ながらのプログラミングとの違い

「え、それってプログラミングと何が違うの？」と思いますよね。ここが本当に大事なので、じっくりいきます。

**昔ながらのプログラミング** は、人間がルールを全部書きます。

```python
# 従来のやり方: 人間がルールを手で書く
def is_cat(mail_or_photo):
    if "耳がとがっている" and "ヒゲがある":
        return "猫"
    else:
        return "猫じゃない"
```

このやり方だと、「じゃあ折れ耳の猫は？」「後ろ姿の猫は？」と、例外が出るたびに人間が `if` を追加し続けなければいけません。世の中の猫のパターンを全部書ききるなんて、とても無理ですよね。

いっぽう **機械学習** は、ルールではなく **「データ（入力）」と「答え（出力）」のペア** をたくさん渡します。すると、コンピュータが「入力から答えを導くルール」を **自分で作り出す**。人間は答え合わせ用のデータを用意するだけでいいんです。

![左は人が手でルールの歯車を組む従来手法、右はデータから自動でパターンが浮かび上がる機械学習の対比図](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-05/figure2.png)

つまり、

- **従来**：人間がルールを書く → コンピュータが実行する
- **機械学習**：人間がデータと答えを渡す → コンピュータがルールを見つける

この「ルールを人が書くか、データから学ぶか」の違いが、機械学習の核心です。深層学習（ディープラーニング）も、この機械学習の中の一種なんですよ（そのつながりは連載の後半でしっかり扱います）。

## 機械学習の3つのタイプ

機械学習は、学び方によって大きく3つに分けられます。**教師あり学習・教師なし学習・強化学習** の3つです。今日いちばん覚えて帰ってほしいのが、この分類。1つずつ、たとえ話でいきましょう。

### ① 教師あり学習（Supervised Learning）

**教師あり学習** は、**「問題」と「正解」がセットになったデータ** で学ぶやり方です。さっきの「これは犬」「これは猫」と正解ラベルを付けて見せる方法が、まさにこれ。

「正解（＝先生の答え）」が付いているから **教師あり** と呼びます。この「正解」のことを **ラベル（label／正解の目印）** と言い、ラベルが付いたデータを **教師データ（訓練データ）** と呼びます。

身近な例をいくつか挙げると、

- 過去のメールに「迷惑メール／普通のメール」のラベルを付けて学習 → 新しいメールを自動で振り分ける
- 過去の写真に「犬／猫」のラベルを付けて学習 → 新しい写真の動物を当てる
- 過去の家のデータ（広さ・駅からの距離）と「実際の販売価格」で学習 → 新しい家の価格を予想する

このあと出てくる連載の中心は、この教師あり学習です。深層学習の入門でいちばんよく使うタイプなので、名前だけでもしっかり覚えておいてください。

![正解ラベル付きのカードが矢印でモデルへ流れ込み、正解を手本に学ぶ教師あり学習の概念図](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-05/figure3.png)

### ② 教師なし学習（Unsupervised Learning）

**教師なし学習** は、その名のとおり **正解ラベルが付いていないデータ** から、コンピュータが自分で構造やグループを見つけるやり方です。「これが正解」という手本がない状態で、データの中の「似ているもの同士」を勝手にまとめてくれるイメージ。

代表例が **クラスタリング（clustering／似たもの同士のグループ分け）** です。たとえば、

- ECサイトの購入履歴から「似た買い方をするお客さん」を自動でグループ分けする
- 大量のニュース記事を、内容が近いもの同士でジャンルごとにまとめる

正解を教えていないのに、データの中の隠れた「かたまり」を見つけてくれる。ここが教師なし学習のおもしろいところです。「教師なし学習の例」としては、このクラスタリングをまず押さえておけば十分です。

![ラベルのない点群が、似たもの同士で自動的に3つの色分けグループにまとまる教師なし学習の概念図](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-05/figure4.png)

### ③ 強化学習（Reinforcement Learning）

**強化学習** は、ちょっと毛色が違います。**「行動して、良ければごほうび（報酬）をもらう」を繰り返して、だんだん上手くなっていく** やり方です。

イメージは、ゲームをプレイして上達していく感じ。うまくいけば得点（報酬）がもらえて、失敗すれば減点される。この試行錯誤をひたすら繰り返すうちに、「こう動けば得点が高い」という行動のコツを自分でつかんでいきます。

- 囲碁や将棋のAIが、対局を何百万回も繰り返して強くなる
- ロボットが歩き方を、転びながら少しずつ覚えていく

正解を1つずつ教えるのではなく、**「結果の良し悪し（報酬）」だけをヒントに** 学ぶ、というのが強化学習の特徴です。

![ロボットが環境で行動し、良い結果に報酬をもらうループを繰り返す強化学習の概念図](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-05/figure5.png)

### 3つの違いをまとめると

「教師あり学習・教師なし学習・強化学習の違い」を、ざっくり一覧にするとこんな感じです。

| タイプ | 与えるもの | 学び方のイメージ | 代表例 |
| --- | --- | --- | --- |
| 教師あり学習 | データ＋**正解ラベル** | 正解を手本にマネる | 迷惑メール判定、価格予測 |
| 教師なし学習 | **ラベルなし**データ | 似たもの同士をまとめる | 顧客のグループ分け |
| 強化学習 | 行動への**報酬** | 試行錯誤で上達 | ゲームAI、ロボット制御 |

この連載でこれから深く掘っていくのは、いちばん左の **教師あり学習** です。残り2つは「そういう仲間もいるんだな」くらいで大丈夫。

## 教師あり学習をもう少しだけ詳しく：分類と回帰

教師あり学習は、さらに **分類（classification）** と **回帰（regression）** の2つに分けられます。「予測したい答えが、種類なのか・数字なのか」で分かれる、と覚えると分かりやすいです。

- **分類**：答えが **カテゴリ（種類）** のとき。「犬か猫か」「迷惑メールかどうか」など、**どのグループに入るか** を当てる。
- **回帰**：答えが **連続した数値** のとき。「この家はいくら？」「明日の気温は何度？」など、**具体的な数字** を当てる。

![左は点を2色の領域に分ける分類、右は点群を通る直線で数値を予測する回帰の対比図](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-05/figure6.png)

「猫か犬か」みたいに **仲間分け** したいなら分類、「価格」みたいに **数字そのもの** を当てたいなら回帰。次回はこの **回帰** を実際に手を動かして体験するので、頭の片隅に置いておいてくださいね。

## 訓練データとテストデータ：なぜ分けるの？

最後に、機械学習を実際にやるときに必ず出てくる、とても大事な考え方を1つ。それが **訓練データ** と **テストデータ** の使い分けです。

手元のデータを、ふつうは2つに分けて使います。

- **訓練データ（training data）**：モデルに **学ばせる（練習させる）** ためのデータ。全体の8割くらい。
- **テストデータ（test data）**：学び終えたモデルの **本当の実力を測る** ためのデータ。残りの2割くらい。

なぜわざわざ分けるのか？　これ、学校の勉強にたとえるとスッと分かります。

練習問題（訓練データ）だけを何度も解いて、答えを丸暗記した子がいたとします。その子に **練習問題とまったく同じ問題** を出せば満点。でも、それでは「本当に理解したのか、ただ暗記しただけなのか」が分かりませんよね。だから **初めて見る問題（テストデータ）** で試すわけです。

機械学習でも同じで、訓練に使ったデータで実力を測ってしまうと、「丸暗記しただけ」のモデルを「賢い！」と勘違いしてしまう。この丸暗記状態を **過学習（overfitting／オーバーフィッティング）** と呼びます。これを見抜くために、学習に使っていない **テストデータ** でちゃんと採点する。この習慣は、この先ずっと付いてまわる超基本ルールです。

![データの山を大きい訓練用と小さいテスト用に分け、学習→評価と流れる図](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-05/figure7.png)

Pythonでは、この「データを訓練用とテスト用に分ける」作業を、たった1行でやってくれる便利な道具があります（今は雰囲気だけ眺めてもらえればOKです）。

```python
# scikit-learn（サイキットラーン／機械学習の定番ライブラリ）を使う例
from sklearn.model_selection import train_test_split

# X = 入力データ（例: 家の広さや駅からの距離）
# y = 正解ラベル（例: 実際の価格）
# test_size=0.2 → 全体の20%をテスト用に取り分ける
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.2, random_state=0  # random_state は結果を固定する種
)

# X_train, y_train で「学習」し、
# X_test, y_test で「実力テスト」する、という流れになる
```

`train_test_split` という名前がもう「訓練とテストを分割する」とそのまま言ってますね。実際に動かすのは次回以降なので、「あ、こうやって分けるんだ」と眺めるだけで十分です。
