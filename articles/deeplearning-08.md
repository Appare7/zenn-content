## 回帰と分類、なにが違うの？

まずは、回帰（regression）と分類（classification）の違いを、しっかりイメージしておきましょう。

回帰は、連続した数値を当てる問題です。

たとえば、

- 家の広さから家賃を予測する
- 勉強時間からテストの点数を予測する

といったものですね。

答えは「12.3万円」とか「78.5点」みたいに、地続きの数字になります。  
数字の目盛りの上を、なめらかに移動できるイメージです。

一方で、分類は「どのグループに入るか」を当てる問題です。

たとえば、

- メールがスパムか否か
- 画像が犬か猫か
- 患者が病気か否か

といったものです。

こちらの答えは、「はい」「いいえ」や「犬」「猫」のような札、つまりラベル（label）になります。  
数直線のどこかを読むというより、「こっちの箱に入れる？それともあっちの箱に入れる？」という仕分けに近いんですね。

特に、答えが2択の分類を二値分類（にちぶんるい／binary classification）と呼びます。  
今日はこの二値分類を扱います。

二値分類では、ラベルを

- 1：陽性・YES
- 0：陰性・NO

のように表すのが定番です。

たとえば「合格か不合格か」なら、

- 合格：1
- 不合格：0

という感じですね。

たとえ話で言うなら、回帰は「体重は何kg？」と体重計の目盛りを読むことです。  
一方、分類は「大人か子どもか？」と仕分けることです。

どちらも予測ではあるのですが、当てたいものの形が違う、というわけです。

## じゃあ線形回帰で分類すればいいの？→うまくいかない

![直線をそのまま使うと0〜1をはみ出すが、S字カーブなら0〜1に収まる、という線形回帰とロジスティック回帰の違いの比較図](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-08/figure2.png)

ここで、ちょっと素朴なアイデアが浮かびます。

「第6回でやった線形回帰を、そのまま分類に使えばいいんじゃない？」

たとえば、線形回帰で

$y = wx + b$

のように計算して、出力が0.5より大きければ1、小さければ0にする。  
なんだかできそうですよね。

でも、これには大きな問題が2つあります。

1つ目は、直線は上にも下にも無限に伸びることです。

分類でほしいのは、「1である確率」のような値です。  
確率なら、必ず0〜1のあいだに収まっていてほしいですよね。

でも、直線をそのまま使うと、出力が 1.8 になったり、-0.5 になったりします。

「合格確率180%です！」  
「スパム確率マイナス50%です！」

と言われても、ちょっと困りますよね。確率としてはありえない値です。

2つ目は、外れ値（はずれち／outlier）の影響を受けやすいことです。

外れ値というのは、ほかのデータから大きく離れた極端なデータのことです。  
線形回帰は直線を引くので、極端なデータが1個あるだけで、直線がぐいっと引っ張られてしまうことがあります。

その結果、「0と1を分ける境界」がずれてしまいやすいんです。

分類でほしいのは、こういう仕組みです。

「どんな入力が来ても、0〜1の“確率”を返してくれる」

そこで登場するのが、シグモイド関数（シグモイドかんすう／sigmoid function）です。

## シグモイド関数：どんな数字も0〜1に押し込む魔法

![横に伸びたS字カーブ。マイナス側は0に、プラス側は1に近づき、真ん中(z=0)で0.5を通るシグモイド関数のグラフ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-08/figure3.png)

シグモイド関数（sigmoid function）は、どんな実数を入れても、必ず0〜1のあいだの値を返してくれる関数です。

グラフの形は、なめらかなS字（エスじ）カーブになります。  
マイナス側では0に近づき、プラス側では1に近づきます。そして真ん中では0.5を通ります。

数式はこうです。

$$
\sigma(z) = \frac{1}{1 + e^{-z}}
$$

ここで、$\sigma$（シグマ）がシグモイドを表しています。  
$e$ は自然対数の底（しぜんたいすうのてい／Napier's constant）、ネイピア数とも呼ばれる数で、約2.718です。

とはいえ、最初は細かい計算よりも、動きのイメージをつかめれば大丈夫です。

- $z$ が大きいほど、出力は1に近づく
- $z$ が小さい、つまりマイナスに大きいほど、出力は0に近づく
- $z = 0$ のとき、出力はちょうど0.5

という感じです。

具体的には、次のような値になります。

- σ(0)=0.5
- σ(2)≈0.88
- σ(-2)≈0.12

つまり、入力が2なら「かなり1っぽい」、入力が-2なら「かなり0っぽい」と見られるわけです。

Pythonで書くとこれだけです。

```python
import numpy as np

def sigmoid(z):
    return 1 / (1 + np.exp(-z))

print(sigmoid(0))   # 0.5
print(sigmoid(2))   # 0.8807...
print(sigmoid(-2))  # 0.1192...
```

ポイントは、この出力を「1である確率」とみなせることです。

たとえば、出力が0.88なら「88%くらい1っぽい」と考えます。  
出力が0.12なら「12%くらい1っぽい」、つまりかなり0側だな、という感じですね。

シグモイド関数は、どんな数字でも0〜1のあいだに丸め込んでくれる、確率変換マシンのような存在なんです。

## ロジスティック回帰の全体像

![入力x→重みをかけて足すz=wx+b→シグモイドで0〜1の確率へ→0.5を境にラベル1か0へ、という処理の流れ図](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-08/figure4.png)

では、いよいよロジスティック回帰（Logistic Regression）の全体像を見ていきましょう。

ロジスティック回帰は、線形回帰とシグモイド関数の合体ワザです。

流れはとてもシンプルで、次の3ステップです。

1. 線形回帰と同じく $z = w \cdot x + b$ を計算する（重み w と バイアス b は学習で決まる）
2. その z をシグモイドに通して確率にする $p = \sigma(z)$
3. p が 0.5 以上なら「1（YES）」、未満なら「0（NO）」と判定する

数式でまとめると、こうなります。

$$
p = \sigma(w \cdot x + b) = \frac{1}{1 + e^{-(w \cdot x + b)}}
$$

ここで $p$ は、「ラベルが1である確率」と考えます。

たとえば、勉強時間から合格するかどうかを予測するなら、

- $p = 0.80$ なら「80%くらい合格しそう」
- $p = 0.20$ なら「20%くらい合格しそう、つまり不合格側っぽい」

という読み方になります。

そして、最後に0.5を境にして、

- $p \geq 0.5$ なら 1
- $p < 0.5$ なら 0

と仕分けます。

ここで、ひとつ紛らわしいポイントがあります。  
ロジスティック「回帰」という名前なのに、やっていることは分類なんです。

なぜ「回帰」という名前が付いているのかというと、いったん確率という“数値”を予測してから、それを0か1の札に変えるからです。  
つまり、内部では数値を出しているけれど、最終的な目的はグループ分け、ということですね。

線形回帰との違いを一言で言うなら、こうです。

線形回帰は、直線をそのまま答えにします。  
ロジスティック回帰は、直線をシグモイドに通して確率にしてから仕分けます。

この「シグモイドを通す」というひと手間が、分類ではとても大事なんです。

## 決定境界（どこで線を引く？）

![2次元の平面上に丸と三角の点が散らばり、その間をまっすぐな線が分けている決定境界のイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-08/figure5.png)

分類では、「どこから先を1にして、どこまでを0にするのか」という境目が必要になります。

この境目を、決定境界（けっていきょうかい／decision boundary）と呼びます。

ロジスティック回帰では、$p = 0.5$ になる場所が境目です。  
そしてシグモイド関数では、$z = 0$ のときに $p = 0.5$ になります。

つまり、

- $z > 0$ なら $p > 0.5$ なので 1側
- $z < 0$ なら $p < 0.5$ なので 0側
- $z = 0$ がちょうど境目

ということです。

入力が1つだけの場合、たとえば「勉強時間だけ」で合格・不合格を予測するなら、境界は「ある1点」になります。

「勉強3.4時間より多ければ合格側、少なければ不合格側」

みたいなイメージですね。

入力が2つある場合、たとえば「身長と体重」から何かを分類するなら、境界は「1本の直線」になります。  
平面上に点が散らばっていて、そのあいだをスパッと線で分ける感じです。

ロジスティック回帰の特徴は、この境界がまっすぐになることです。  
入力が1つなら点、2つなら直線、もっと増えると平面のようなものになります。

たとえるなら、机の上に散らばったカードを、定規で引いた一本の線で「こっち側」「あっち側」に分けるような感じです。  
とても素直な分類器なんですね。

## 損失はMSEじゃダメ？→交差エントロピー損失

![正解の確率が1に近いほど損失0、0に近づくほど損失が急増する -log(p) のカーブのイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-08/figure6.png)

モデルを学習するには、「予測がどれだけ間違っているか」を測る必要がありました。  
これが損失関数（loss function）でしたね。第7回の大事な話です。

回帰では、MSE（平均二乗誤差／mean squared error）を使いました。  
予測と正解の差を二乗して平均する、というものでした。

では、分類でもMSEを使えばいいのでしょうか？

実は、分類でMSEを使うと、シグモイドと相性が悪くて学習が進みにくくなることがあります。  
勾配（こうばい／gradient）がほぼ0になって、パラメータがうまく動かなくなる場所ができるんです。

そこで、分類でよく使われるのが交差エントロピー損失（こうさエントロピーそんしつ／cross entropy loss）です。  
二値分類版は、バイナリ交差エントロピー（binary cross entropy）とも呼びます。交差エントロピー誤差という言い方もします。

データ1個ぶんの式は、こうです。

$$
L = -\left[\, y \log(p) + (1-y)\log(1-p) \,\right]
$$

ここで、

- $y$ は正解ラベルです。1か0です。
- $p$ はモデルが予測した「1である確率」です。
- $L$ は損失です。

この式、最初は少しこわく見えるかもしれません。  
でも、二値分類では $y$ が1か0なので、実はかなり見やすい形になります。

正解 $y$ が 1 のとき、式の後半は消えます。  
なぜなら $(1-y)$ が0になるからです。

そのため、効くのは前半の $-\log(p)$ だけです。

このとき、

- 予測 $p$ が1に近い → $-\log(p)$ はほぼ0 → 損失は小さい
- 予測 $p$ が0に近い → $-\log(p)$ は無限大に近づく → 損失が激増

となります。

つまり、正解が1なのに「0っぽいです！」と自信満々に外すと、ものすごく怒られるわけです。

逆に、正解 $y$ が 0 のときは、前半が消えて後半の $-\log(1-p)$ だけが効きます。

この場合も同じです。

- 正解が0で、予測 $p$ が0に近い → よい予測なので損失は小さい
- 正解が0なのに、予測 $p$ が1に近い → 大外しなので損失が大きい

つまり交差エントロピー損失は、「自信満々で間違えると、めちゃくちゃ怒られる」損失なんです。

これは直感的ですよね。  
「たぶん合格かな……？」と弱く外すより、「絶対合格です！」と強く言い切って外したほうが、反省してほしい感じがします。

全データでは、この損失の平均をとります。

Pythonで書くと、次のようになります。  
`log(0)` を避けるために、予測値を少しだけクリップしています。

```python
def bce_loss(y_true, y_pred):
    eps = 1e-7                                  # log(0)で無限大にならないための保険
    y_pred = np.clip(y_pred, eps, 1 - eps)
    return -np.mean(y_true * np.log(y_pred) + (1 - y_true) * np.log(1 - y_pred))

print(bce_loss(np.array([1, 0]), np.array([0.9, 0.1])))  # 0.105 くらい（よい予測→損失小）
print(bce_loss(np.array([1, 0]), np.array([0.1, 0.9])))  # 2.30 くらい（大外し→損失大）
```

よい予測では損失が小さく、大外しでは損失が大きくなっていますね。  
この「間違い方の強さ」をちゃんと見てくれるところが、分類に向いているポイントなんです。

## シグモイドの微分と、学習のしくみ（勾配）

![シグモイドのS字カーブと、その傾き(微分)が真ん中で最大・両端で0に近づくベル型カーブのイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-08/figure7.png)

学習の流れは、前回と同じです。

勾配降下法（gradient descent）を使って、損失が小さくなる向きへ、重み $w$ とバイアス $b$ を少しずつ動かしていきます。

ここで大事になるのが微分（びぶん／derivative）です。  
微分は、ざっくり言うと「いま、その場所でどっち向きにどれくらい傾いているか」を教えてくれるものです。

山登りでたとえるなら、足元の地面がどちらに下っているかを調べる感じです。  
損失を小さくしたいので、モデルはその下り坂の方向へ少しずつ進んでいきます。

ここで嬉しい事実があります。

シグモイドの微分は、とてもきれいな形になります。

$$
\sigma'(z) = \sigma(z)\,(1 - \sigma(z))
$$

自分自身 $\sigma(z)$ を使って、微分まで表せてしまうんです。  
なんだか、よくできていますよね。

さらに、交差エントロピー損失とシグモイドを組み合わせると、パラメータの勾配、つまり更新に使う量がびっくりするほどシンプルになります。

$$
\frac{\partial L}{\partial w} = (p - y)\,x, \qquad \frac{\partial L}{\partial b} = (p - y)
$$

ここで主役になっているのは、$p - y$ です。

つまり、「予測 $p$ と正解 $y$ の差」ですね。

これ、かなり自然です。

たとえば正解が1で、予測が0.9なら、

- $p - y = 0.9 - 1 = -0.1$

なので、差は小さいです。  
けっこう当たっているので、パラメータも少しだけ動けばよさそうですよね。

一方、正解が1なのに予測が0.1なら、

- $p - y = 0.1 - 1 = -0.9$

なので、差は大きいです。  
大きく外しているので、しっかり修正したい場面です。

このように、予測が当たっていれば差は小さく、外していれば差は大きくなります。  
そして、その差がそのまま更新の主役になります。

この美しさが、ロジスティック回帰がニューラルネットの1個ぶんの部品、つまりニューロン（neuron）とほぼ同じ形になっている理由でもあります。  
ここは今は「へえ、深層学習にもつながっているんだな」くらいで大丈夫です。次につながる大事な橋だと思ってください。

## Pythonで動かしてみる（勉強時間→合格予測）

![横軸=勉強時間、縦軸=合格確率。データ点(不合格0/合格1)の間をS字カーブが通り、確率が上がっていく作例のイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-08/figure8.png)

では、NumPyだけでロジスティック回帰をスクラッチ実装してみましょう。

例題は、「勉強時間から合格するかどうかを予測する」です。

勉強時間が少ないと不合格、勉強時間が多いと合格、という小さなデータを使います。  
モデルがちゃんと「勉強するほど合格しやすい」を学べるか、見てみましょう。

```python
import numpy as np

def sigmoid(z):
    return 1 / (1 + np.exp(-z))

# 勉強時間（時間）と、その結果の合格ラベル（1=合格, 0=不合格）
X = np.array([1., 2., 3., 4., 5., 6.])
y = np.array([0,  0,  0,  1,  1,  1])

w, b = 0.0, 0.0   # パラメータの初期値
lr = 0.1          # 学習率

for step in range(2000):
    p = sigmoid(w * X + b)     # いまのw,bでの予測確率
    dw = np.mean((p - y) * X)  # wの勾配（p - y が主役）
    db = np.mean(p - y)        # bの勾配
    w -= lr * dw               # 損失が下がる向きへ更新
    b -= lr * db

print("w =", round(w, 3), " b =", round(b, 3))          # w = 2.264  b = -7.704
print("2時間の合格確率:", round(float(sigmoid(w*2 + b)), 3))  # 0.04
print("5時間の合格確率:", round(float(sigmoid(w*5 + b)), 3))  # 0.974
```

結果を見ると、学習後は

- $w \approx 2.26$
- $b \approx -7.70$

となります。

そして予測は、

- 2時間しか勉強していないと、合格確率は約4%
- 5時間勉強していると、合格確率は約97%

になります。

ちゃんと「勉強するほど合格しやすい」という関係を学べていますね。

決定境界、つまり確率が50%になる場所も見てみましょう。  
確率50%は $z = 0$ の場所なので、

$w x + b = 0$

となる $x$ が境目です。

今回の値では、だいたい

$x = 7.70 / 2.26 \approx 3.4$

なので、決定境界はだいたい3.4時間あたりです。

つまり、

- 3.4時間より少なければ不合格側
- 3.4時間より多ければ合格側

という判断をするモデルになった、というわけです。

実務では、scikit-learn（サイキットラーン）というライブラリの `LogisticRegression` を使えば、ロジスティック回帰は数行で書けます。  
でも今日は、中身を理解するために、あえて手で書いてみました。

## まとめ

今回は、分類の基本であるロジスティック回帰を見てきました。

ポイントをまとめると、こんな感じです。

- 分類（classification）は、データをグループ分けする問題
- 二値分類（binary classification）は、答えが2択の分類
- 二値分類では、ラベルを 1 と 0 で表すのが定番
- 線形回帰をそのまま使うと、出力が0〜1をはみ出してしまい、確率として扱いにくい
- シグモイド関数を使うと、どんな数字も0〜1のあいだに変換できる
- ロジスティック回帰は、$z = wx + b$ をシグモイドに通して確率にする
- その確率が0.5以上なら1、未満なら0に仕分ける
- 分類の損失には、交差エントロピー損失を使う
- 交差エントロピー損失は、大外しほど激しく怒ってくれる損失
- シグモイドと交差エントロピーを組み合わせると、勾配は $(p-y)x$ ととてもシンプルになる
- NumPyだけでも、ロジスティック回帰を実際に学習できた

ロジスティック回帰という名前はちょっと強そうですが、中身はかなり素直でしたよね。

「直線でスコアを出す」  
「シグモイドで確率にする」  
「0.5で分ける」  
「外れたぶんだけ直す」

この流れがつかめていれば、もう大丈夫です。

そしてこの考え方は、これから出てくるニューラルネットワークにもつながっていきます。  
ひとつずつ積み上がってきていますね。いい感じです。
