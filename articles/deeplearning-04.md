## そもそもNumPyって何者？なぜ使うの？

Pythonにも標準の **リスト（`[1, 2, 3]` みたいなやつ）** がありますよね。「じゃあリストで計算すればいいじゃん」と思うところですが、リストで大量の数値計算をやると、実は **すごく遅い** し **書くのも面倒** なんです。

たとえば「リストの全要素を2倍したい」だけでも、素のPythonだと `for` ループを回す必要があります。要素が100万個あったら、100万回ループ。これがニューラルネットのように何億回も計算する世界だと、致命的に遅い。

そこで **NumPy** の出番です。NumPyは数値がぎっしり詰まった専用の入れ物 **ndarray（エヌディーアレイ／N-dimensional array、多次元配列）** を用意していて、

- **まとめてドカッと計算できる**（ループを自分で書かなくていい）
- **内部がC言語で書かれていて速い**

という2つの強みがあります。まずはインストールして読み込むところから。

```python
# ターミナルで一度だけ: pip install numpy
import numpy as np   # np という短い名前で呼ぶのが世界共通の慣習
```

以降、`np.〜` と出てきたら「NumPyの機能を呼んでいるんだな」と思ってください。

## まずは配列を作ってみる（1次元・2次元）

NumPyの基本の入れ物、ndarrayを作ってみましょう。Pythonのリストを `np.array()` に渡すだけです。

```python
import numpy as np

# 1次元配列（＝ベクトル。数字を一列に並べたもの）
v = np.array([1, 2, 3])
print(v)          # [1 2 3]

# 2次元配列（＝行列。数字を縦横の表に並べたもの）
A = np.array([[1, 2, 3],
              [4, 5, 6]])
print(A)
# [[1 2 3]
#  [4 5 6]]
```

第1回で見た **ベクトル（数字の一列）** と **行列（数字の表）** が、そのままコードになりました。2次元配列は「リストの中にリストを入れる」と作れます。内側の1つ1つが「行」になるイメージですね。

![光る小さな正方形セルが行と列に整然と並び、1行と1列がそれぞれteal・青でハイライトされた2次元配列の概念図](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-04/figure2.png)

よく使う「配列の作り方」も一緒に覚えておくと便利です。

```python
np.zeros((2, 3))   # 全部0の、2行3列の行列
np.ones((2, 3))    # 全部1の、2行3列の行列
np.arange(0, 6)    # [0 1 2 3 4 5]（連番。range のNumPy版）
np.random.rand(2, 3)  # 0〜1の乱数で埋めた2行3列（重みの初期化などで多用）
```

## 形（shape）と次元 ── 配列の「住所表示」

NumPyを使ううえで、**いちばん大事なのに初心者がいちばん軽視しがちな概念** が、この **形（shape／シェイプ）** です。断言しますが、深層学習のバグの大半は「形が合っていない」ことが原因です。ここを最初に押さえておくと、この先の消耗がかなり減ります。

形は `.shape` で確認できます。

```python
v = np.array([1, 2, 3])
A = np.array([[1, 2, 3],
              [4, 5, 6]])

print(v.shape)   # (3,)    → 要素3つの1次元配列
print(A.shape)   # (2, 3)  → 2行3列の2次元配列
print(A.ndim)    # 2       → 次元の数（何重にネストしているか）
```

`(2, 3)` は「2行3列」を表します。読み方は **「外側から内側へ」**。`A` はまず「2つの行」があって、その各行が「3つの数字」を持っている、というわけです。`.ndim` は次元数（軸の数）で、ベクトルは1、行列は2、後の回で出てくる画像データなどは3や4になります。

![1列のセル(1次元)・平らな格子(2次元)・格子を積んだ立方体(3次元)をロボットが虫めがねで確認し、2つの形が揃うかチェックしているイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-04/figure8.png)

**「困ったら `print(x.shape)`」** ——これは深層学習エンジニアが一生使う口ぐせです。今から癖にしておきましょう。

## 足し算・スカラー倍 ── 要素ごとにまとめて計算

ここからが、NumPyの気持ちよさの本番です。同じ形の配列どうしなら、`+` や `-` を書くだけで **全要素が一気に** 計算されます。ループは一切いりません。

```python
A = np.array([[1, 2, 3],
              [4, 5, 6]])
B = np.array([[10, 20, 30],
              [40, 50, 60]])

print(A + B)
# [[11 22 33]
#  [44 55 66]]   ← 同じ位置どうしを足しただけ
```

数字1個（**スカラー**）を掛けたり足したりもできます。これは配列の全要素に効きます。

```python
print(A * 2)
# [[ 2  4  6]
#  [ 8 10 12]]   ← 全部2倍

print(A + 100)
# [[101 102 103]
#  [104 105 106]]  ← 全部に100を足す
```

「行列に数字を1個掛けたら、中身が全部その倍になる」。第1回で紙の上でやった **スカラー倍** が、そのままコードで再現できました。

![同じ形の格子2枚が重なって足し合わされる様子と、1枚の格子全体が均一に明るくなる（スカラー倍）様子を並べたイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-04/figure3.png)

## 2種類の「掛け算」：要素ごとの積（`*`）と行列積（`@`）

ここが今回いちばんの **ハマりどころ** であり、いちばん大事なポイントです。NumPyの「掛け算」には **まったく別物の2種類** があります。第1回でも少し触れましたが、コードでしっかり体感しておきましょう。

**① 要素ごとの積（アダマール積）── 記号は `*`**

同じ位置の数字どうしを、ただ掛けるだけ。結果の形は元と同じです。

```python
A = np.array([[1, 2],
              [3, 4]])
B = np.array([[5, 6],
              [7, 8]])

print(A * B)
# [[ 5 12]
#  [21 32]]   ← 1×5, 2×6, 3×7, 4×8 とその場で掛けただけ
```

**② 行列積 ── 記号は `@`（または `np.matmul`）**

第1回でやった「左の行と右の列を、掛けて足す」あの計算です。要素ごとの積とはまったく違う結果になります。

```python
print(A @ B)
# [[19 22]
#  [43 50]]
```

同じ `A`, `B` なのに、`*` と `@` で結果が全然違いますよね。この違いを絵にするとこんなイメージです。

![左は2つの格子をセルごとに重ねる要素ごとの積、右は片方の行がもう片方の列を掃くように進む行列積、という2種類の掛け算の対比図](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-04/figure4.png)

深層学習で「掛ける」といったら、ほぼ **行列積（`@`）** のことを指します。ニューラルネットの「入力に重みを掛けて足す」があの行列積そのものだからです。**`*` と `@` を取り違えると、エラーも出ずに間違った答えが返る**——という一番こわいバグになりがちなので、ここは指に覚えさせてください。

## 行列積の仕組みをコードで確かめる（内積）

行列積の中身は、突きつめると **内積（ないせき／dot product）** の集まりです。内積は「2つのベクトルを、掛けて足して1つの数にする」計算のこと。

```python
a = np.array([1, 2, 3])
b = np.array([4, 5, 6])

print(np.dot(a, b))   # 32
# 計算： 1×4 + 2×5 + 3×6 = 4 + 10 + 18 = 32
```

行列積は、この内積を「左の各行 × 右の各列」の全組み合わせについて計算して、表に並べ直したもの。だから **左の列数と右の行数が一致していないと計算できません**。ここが形（shape）の話とつながってきます。

```python
X = np.array([[1, 2, 3],
              [4, 5, 6]])        # 形 (2, 3)
W = np.array([[1, 0],
              [0, 1],
              [1, 1]])           # 形 (3, 2)

print((X @ W).shape)   # (2, 2)
# 左の列数(3) と 右の行数(3) が一致 → 計算できる。結果は (2, 2)
```

コツは **「内側の数字が一致し、外側の数字が結果の形になる」**。`(2, 3) @ (3, 2)` なら、内側の `3` と `3` が一致していればOKで、外側の `2` と `2` が答えの形になる、というルールです。

![光る横1行と縦1列が1つの明るいノードで交わり、隣で歯車が噛み合う——行×列の内積が行列積の1マスを作る仕組みのイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-04/figure5.png)

## ブロードキャスト ── 形が違っても計算できる仕組み

前回予告した **ブロードキャスト（broadcasting）** です。これはNumPyの超便利機能で、**形が違う配列どうしでも、NumPyが自動で“形を合わせて”計算してくれる** というもの。

たとえば「行列の各行に、同じベクトルを足したい」とき。ふつうに考えると形が合わなくてエラーになりそうですが……

```python
M = np.array([[1, 2, 3],
              [4, 5, 6]])   # 形 (2, 3)
v = np.array([10, 20, 30])  # 形 (3,)

print(M + v)
# [[11 22 33]
#  [14 25 36]]   ← v が各行に自動でコピーされて足された！
```

`v` はたった1本のベクトルなのに、`M` の **2つの行それぞれに、勝手に複製されて** 足されました。これがブロードキャストです。小さいほうの配列が、大きいほうに合わせて「引き伸ばされる」イメージ。

![小さな1本のセル列が自動的に複製・拡大して大きな格子全体を覆っていく、ブロードキャストのイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-04/figure6.png)

ルールはシンプルで、**形を末尾（右側）から見比べて、「数字が同じ」か「どちらかが1」なら引き伸ばして揃える**、というもの。スカラー倍（`A * 2`）も、実は「`2` を全要素に引き伸ばした」ブロードキャストの一種なんです。深層学習では「バイアス（各ニューロンに足す下駄）を全データに足す」といった場面で、これでもかと登場します。今は「形が違っても、いい感じに揃えてくれることがある」くらいの感覚でOKです。

## 転置（`.T`）── 行と列を入れ替える

最後に、地味だけど超頻出の **転置（てんち／transpose）**。行と列をパタンと入れ替える操作で、`.T` を付けるだけです。

```python
A = np.array([[1, 2, 3],
              [4, 5, 6]])   # 形 (2, 3)

print(A.T)
# [[1 4]
#  [2 5]
#  [3 6]]        ← 形 (3, 2) に入れ替わった
```

「2行3列」が「3行2列」にひっくり返りました。これ、さっきの行列積で「形が合わない！」というときに、**片方を転置すると形が揃って計算できる**、という救済策としてめちゃくちゃ使います。

![長方形の格子が対角線を軸に反転し、行が列に入れ替わる転置のイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-04/figure7.png)

## よくあるエラーと、その読み解き方

NumPy初心者が必ず出会うエラーを、先回りして紹介しておきます。恐れることはありません。ほとんどが **「形が合ってないよ」** という同じ話です。

```python
X = np.array([[1, 2, 3],
              [4, 5, 6]])   # (2, 3)
W = np.array([[1, 2],
              [3, 4]])      # (2, 2)

X @ W
# ValueError: matmul: ... (size 3 is different from 2)
```

これは行列積で「左の列数(3)と右の行数(2)が合わない」と言われています。エラー文に出てくる数字を見て、`.shape` で形を確かめれば、たいてい原因はすぐ分かります。**エラーが出たら、まず関係する配列の `.shape` を全部 `print` する**。これが最速の解決法です。困ったら形を見る——今回いちばん持ち帰ってほしい習慣です。

## まとめ

今回の要点をおさらいします。

- **NumPy** は多次元配列（ndarray）をまとめて高速に計算できるライブラリ。`np.array()` で配列を作る。
- **形（shape）** は配列の「住所表示」。`(2, 3)` は2行3列。**困ったら `print(x.shape)`** が合言葉。
- 足し算・スカラー倍は、書くだけで **全要素にまとめて** 効く。ループは不要。
- 掛け算は2種類。**要素ごとの積は `*`、行列積は `@`**。深層学習で主役なのは `@` のほう。取り違え注意。
- 行列積は **内積の集まり**。「内側の数字が一致、外側が結果の形」。
- **ブロードキャスト** は、形が違っても自動で揃えて計算してくれる仕組み。バイアスを足すときなどに大活躍。
- **転置（`.T`）** は行と列の入れ替え。形合わせの救済策として頻出。

紙の上の数式が、コードで動くものに変わった感覚を持ってもらえたら大成功です。ここで手に入れた「配列を作る・足す・掛ける・形を合わせる」は、この先ずっと使う土台になります。
