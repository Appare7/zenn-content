## 過学習ってなに？「丸暗記した優等生」のたとえ

過学習（かがくしゅう／overfitting）とは、訓練データに合わせすぎて、新しいデータにうまく対応できなくなることです。

たとえば、過去問だけを丸暗記した学生を想像してみてください。

過去問、つまり訓練データでは満点が取れます。  
でも、本番で少しだけ問題がひねられると、急に解けなくなってしまう。

これは「理解」しているのではなく、「暗記」してしまった状態ですよね。

機械学習のモデルでも同じことが起きます。訓練データに含まれる細かいクセや、たまたまのブレまで覚えてしまうと、新しいデータに弱くなります。

ここで大事なのが、モデルの複雑さです。

モデルが複雑すぎる、つまり表現力がありすぎると、データのノイズ（のいず／noise、たまたまのブレ）まで覚えてしまいます。  
本当は「全体の傾向」をつかんでほしいのに、「この点はちょっと上、この点はちょっと下」みたいな細かいところまで追いかけてしまうんですね。

曲線でイメージすると、過学習したモデルは、点を全部通ろうとしてぐにゃぐにゃ曲がる感じです。  
がんばりすぎなんです。えらいんだけど、そこまで覚えなくていいよ、という状態です。

逆に、モデルが簡単すぎて、過去問すらまともに解けない状態もあります。  
これを未学習（みがくしゅう／underfitting）といいます。

つまり、ざっくり言うとこうです。

- 未学習：簡単すぎて、訓練データにも合わない
- ちょうどいい：訓練データにも、新しいデータにもそこそこ合う
- 過学習：訓練データに合わせすぎて、新しいデータに弱い

目指したいのは、もちろん「ちょうどいい」場所です。

![左から未学習・ちょうどいい・過学習。右へいくほど複雑で、点を全部通ろうとぐにゃぐにゃに](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-09/figure2.png)

## 訓練誤差と検証誤差でサインを見抜く

過学習のやっかいなところは、「訓練データの成績」だけ見ていると気づけないことです。

訓練誤差（くんれんごさ／training error）とは、訓練に使ったデータでの間違いの大きさです。  
モデルが学習中に何度も見ているデータなので、ここでの成績はだんだん良くなっていきます。

一方で、検証誤差（けんしょうごさ／validation error）とは、学習に使っていない別データでの間違いの大きさです。  
こちらのほうが、本当の実力に近いです。

ここ、かなり大事です。

学習を進めるほど、訓練誤差はどんどん下がっていきます。  
でも、検証誤差はあるところまでは下がるものの、途中から逆に上がり始めることがあります。

この「検証誤差がいちばん低いところ」が、ちょうどいいポイントです。

イメージとしては、検証誤差がU字のカーブを描く感じです。  
最初は学習が進んで良くなる。  
でも、途中から訓練データの細かいクセを覚えすぎて、本番力が落ちていく。

たとえるなら、過去問の点数だけは上がっているのに、模試の点数が下がり始めた状態です。  
「あれ、これは暗記に走ってるかも？」というサインなんですね。

なので、モデルを見るときは、訓練誤差だけではなく、検証誤差も一緒に見ることがとても大切です。

![訓練誤差は下がり続けるのに、検証誤差はU字を描く。この底がちょうどいいポイント](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-09/figure3.png)

## データは3つに分ける：訓練・検証・テスト

では、どうやって訓練誤差や検証誤差を見るのでしょうか。

そのために、データを分けます。

訓練に使ったデータで実力を測るのは、答えを見ながらテストするようなものです。  
ちょっと言い方を強めると、カンニングみたいなものですね。

だから、学習に使うデータと、実力を測るデータは分けておきます。

よく使う分け方は、次の3つです。

- 訓練データ（くんれんでーた／training data）：モデルに学習させる、つまりパラメータ（ぱらめーた／parameter、モデルが学習で調整する値）を調整するためのデータ
- 検証データ（けんしょうでーた／validation data）：学習の途中で「今どのくらいの実力？」を確かめるためのデータ。モデルの設定、たとえば層の数や正則化の強さなどを選ぶために使う
- テストデータ（てすとでーた／test data）：最後の最後に一度だけ使う、本番想定の実力チェック用データ。最後まで絶対に学習に使わない・のぞき見しない

ここで出てきた「モデルの設定」は、ハイパーパラメータ（はいぱーぱらめーた／hyperparameter、学習前に人が決める設定値）と呼ばれます。  
たとえば、あとで出てくる正則化の強さなどがそうです。

割合の目安としては、だいたい次のように分けることがあります。

- 訓練 60%
- 検証 20%
- テスト 20%

もちろん、これは絶対ではありません。あくまで目安です。

こうやってデータを取り分ける方法を、ホールドアウト法（ほーるどあうとほう／holdout、データを分けて評価する方法）といいます。

また、データが少ないときには、交差検証（こうさけんしょう／cross-validation）という方法もあります。  
これは、分け方を変えて何回も試して、その平均を見る方法です。ここでは名前だけ知っておけばOKです。

コードで書くと、たとえばこんな感じです。  
scikit-learn（さいきっとらーん／scikit-learn、機械学習用ライブラリ）の `train_test_split` を使って、60/20/20に分けています。

```python
from sklearn.model_selection import train_test_split

# まず「訓練＋検証」と「テスト」に分ける（テストは最後まで触らない）
X_trainval, X_test, y_trainval, y_test = train_test_split(
    X, y, test_size=0.2, random_state=0)

# 次に「訓練」と「検証」に分ける
X_train, X_val, y_train, y_val = train_test_split(
    X_trainval, y_trainval, test_size=0.25, random_state=0)

# これで だいたい 訓練60% / 検証20% / テスト20%
print("訓練:", len(X_train), "検証:", len(X_val), "テスト:", len(X_test))
# → 訓練: 24 検証: 8 テスト: 8  （全体40個のデータのとき）
```

ポイントは、テストデータは最後まで触らないことです。  
検証データは途中で使いますが、テストデータは最後の最後。本番リハーサルみたいなものですね。

![データは訓練・検証・テストの3つに分ける。テストは最後に一度だけ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-09/figure4.png)

## 正則化：モデルを「ほどよく鈍感」にする

過学習を抑える代表的な工夫が、正則化（せいそくか／regularization）です。

正則化は、モデルを「ほどよく鈍感」にするための方法だと思ってください。

モデルが複雑になりすぎると、重み、つまりパラメータが極端に大きくなりがちです。  
大きい重みは、データのちょっとしたブレに敏感に反応します。

その結果、曲線がぐにゃぐにゃになって、訓練データにはよく合うけれど、新しいデータには弱いモデルになってしまいます。  
これが過学習でしたね。

そこで、「重みが大きくなりすぎたらペナルティ（ぺなるてぃ／penalty、罰）を与える」ようにします。

たとえるなら、暴れる曲線に手綱をかける感じです。  
「そんなに細かく曲がらなくていいよ。もう少しなめらかにいこうね」とブレーキをかけるわけです。

![正則化は、暴れる曲線に手綱をかけてなめらかにするイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-09/figure5.png)

ふつうの損失（そんしつ／loss）に、ペナルティ項を足します。

L2正則化（えるつーせいそくか／L2 regularization、別名 weight decay／ウェイト減衰）の損失は、次の形です。  
$n$ はデータ数、$y_i$ は正解、$\hat{y}_i$ は予測、$w_j$ は重み、$\lambda$（ラムダ）は正則化の強さを決める値です。  
$\lambda$ が大きいほど、重みを強く抑えます。

$$ L(w) = \frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2 + \lambda \sum_{j} w_j^2 $$

L2正則化は、重みの「2乗の合計」を足します。  
これによって、重みを全体的に小さめに保つ働きがあります。

一方で、L1正則化（えるわんせいそくか／L1 regularization）は「絶対値の合計」を足す形です。

$$ L(w) = \frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2 + \lambda \sum_{j} |w_j| $$

L1とL2の違いをざっくり言うと、こうです。

- L1：いらない重みをきっぱり0にしやすい。使う特徴を絞る感じ。スパース（すぱーす／sparse、0が多くてすっきりした状態）になりやすい
- L2：全部の重みをまんべんなく小さくする

どちらも、「重みを暴れさせない」という点では同じです。

ただし、$\lambda$ が大きすぎると、今度は抑えすぎになります。  
手綱を引きすぎて、モデルが自由に学べなくなるんですね。

その結果、未学習になってしまうことがあります。  
なので、正則化の強さは検証データで探します。ここで、さっきの「訓練・検証・テストに分ける理由」がつながってきます。

![L1はいらない重みを0にしやすく、L2は全体をまんべんなく小さくする](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-09/figure6.png)

## コードで体感：正則化ありなしで過学習を比べる

では、コードで正則化の効果を見てみましょう。

ここでは、15次の多項式（たこうしき／polynomial）で、わざと複雑なモデルを作ります。  
そして、正則化なし、つまりふつうの線形回帰（せんけいかいき／Linear Regression）と、正則化ありのRidge（りっじ／Ridge、リッジ回帰）を比べます。

```python
import numpy as np
from sklearn.linear_model import LinearRegression, Ridge
from sklearn.preprocessing import PolynomialFeatures
from sklearn.pipeline import make_pipeline
from sklearn.model_selection import train_test_split

# サイン波にノイズをのせた、ちょっとバラついたデータを作る
rng = np.random.default_rng(0)
X = np.linspace(-3, 3, 40).reshape(-1, 1)
y = np.sin(X).ravel() + rng.normal(0, 0.3, size=40)

# 訓練データとテストデータに分ける
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=0)

# 15次の多項式で、わざと複雑なモデルにする
no_reg   = make_pipeline(PolynomialFeatures(degree=15), LinearRegression())   # 正則化なし
with_reg = make_pipeline(PolynomialFeatures(degree=15), Ridge(alpha=1.0))      # L2正則化あり

for name, model in [("正則化なし", no_reg), ("正則化あり(L2)", with_reg)]:
    model.fit(X_train, y_train)
    tr = model.score(X_train, y_train)  # 訓練データでのR2スコア
    te = model.score(X_test, y_test)    # テストデータでのR2スコア
    print(f"{name}: 訓練R2={tr:.2f} / テストR2={te:.2f}")

# 出力:
# 正則化なし: 訓練R2=0.98 / テストR2=0.80   ← 訓練は完璧に近いのにテストが低い＝過学習
# 正則化あり(L2): 訓練R2=0.95 / テストR2=0.90 ← 訓練を少し我慢してテストが上がった
```

出力を見ると、こうなっています。

- 正則化なし: 訓練R2=0.98 / テストR2=0.80
- 正則化あり(L2): 訓練R2=0.95 / テストR2=0.90

R2（あーるつー／R-squared、決定係数）は、1に近いほど良い指標です。

正則化なしのモデルは、訓練R2が0.98です。ほぼ完璧ですね。  
でも、テストR2は0.80で止まっています。

これは典型的な過学習のサインです。  
訓練データにはすごく合っているけれど、新しいデータではそこまで強くない、ということです。

一方、正則化ありのモデルは、訓練R2が0.95に少し下がっています。  
でも、テストR2は0.90に上がっています。

つまり、訓練の点をちょっと我慢した代わりに、本番の点が上がったわけです。

これが正則化のごほうびです。  
「訓練データで満点を取ること」よりも、「新しいデータでちゃんと当たること」のほうが大事なんですね。

ちなみに、コードの `alpha`（あるふぁ／alpha）は、Ridge の正則化の強さです。  
数式で出てきた $\lambda$ にあたるものだと思ってください。

## もうひとつの武器：ドロップアウトと早期終了

過学習を防ぐ方法は、正則化だけではありません。

深層学習でよく出てくる方法として、ドロップアウト（どろっぷあうと／dropout）があります。

ドロップアウトは、学習のたびに、ニューラルネットのニューロン（にゅーろん／neuron、信号を受け取って次に渡す小さな部品）やノード（のーど／node、ネットワーク内の点）をランダム（らんだむ／random、無作為）に一部お休みさせる方法です。

毎回、全員フルメンバーで練習するのではなく、一部のメンバーを欠席させて練習するチームを想像してみてください。

そうすると、特定の人だけに頼りすぎず、チーム全体で力をつけようとしますよね。  
ニューラルネットでも同じで、特定のニューロンに頼りすぎるのを防いで、過学習を抑えます。

ドロップアウトは、深層学習で特によく使う方法です。  
次の章以降でニューラルネットを作るときに、また出てきます。

![ドロップアウトは、一部のニューロンをランダムにお休みさせる](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-09/figure7.png)

もうひとつが、早期終了（そうきしゅうりょう／early stopping）です。

早期終了は、検証誤差が上がり始めたら、そこで学習をストップする方法です。  
さっき見たU字カーブの底で止めるイメージですね。

学習を続ければ続けるほど、訓練誤差は下がります。  
でも、検証誤差が上がり始めたなら、それ以上は「本番に弱くなる方向」に進んでいるかもしれません。

だったら、いちばん良さそうなところで止めよう、というわけです。

とてもシンプルですが、強力な考え方です。  
「がんばりすぎる前に止める」というのも、過学習対策では大事なんですね。

![早期終了は、検証誤差が上がり始める前に学習を止める](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-09/figure8.png)

## まとめ

今回は、過学習と正則化、そしてデータの分け方について学びました。

大事なところを振り返っておきましょう。

- 過学習（overfitting）は、訓練データを「丸暗記」してしまい、新しいデータに弱くなること
- 訓練データだけ成績が良くて、検証データやテストデータで悪いなら、過学習のサイン
- データは、訓練・検証・テストに分ける
- テストデータは、最後の最後に一度だけ使う。本番力チェック用なので、のぞき見しない
- 正則化（regularization）は、重みを暴れさせないための工夫
- L2正則化は、重みを全体的に小さくしやすい
- L1正則化は、いらない重みを0にしやすく、スパースになりやすい
- ドロップアウトは、一部のニューロンをランダムに休ませて、頼りすぎを防ぐ方法
- 早期終了は、検証誤差が悪くなり始めたところで学習を止める方法

モデルに完璧に暗記させるより、「ほどよく」学ばせること。  
これが、賢いモデルを作るコツなんです。

ちょっと人間の勉強にも似ていますよね。過去問を覚えるだけではなく、考え方をつかむ。  
機械学習でも、そこがとても大切です。
