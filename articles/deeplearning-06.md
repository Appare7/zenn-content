## 線形回帰ってなに？　いちばんシンプルな「予測」

まずは具体例から。あなたが中古住宅の値段を予想したいとします。手元には「家の広さ」と「実際の売値」のデータがいくつかあります。

- 40㎡ → 2000万円
- 60㎡ → 2800万円
- 80㎡ → 3600万円
- ……

なんとなく「広いほど高い」という関係が見えますよね。この関係を **1本の直線** で表してしまおう、というのが線形回帰の発想です。直線さえ引ければ、データにない「70㎡の家っていくらくらい？」にも、その直線の上を見れば答えられます。これが **予測** です。

つまり線形回帰とは、

> **入力（家の広さ）と出力（値段）の関係を1本の直線で表して、まだ知らない入力に対する出力を予測する仕組み**

のこと。ものすごくシンプルでしょう？　でも「データからいい直線を見つける」というこの作業こそ、深層学習まで続く「学習」の原型なんです。

![家の広さを入力すると直線を通って予測価格が出てくる流れの図](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-06/figure2.png)

## 直線の正体は `y = a x + b`

中学の数学を思い出してください。直線はこの式で書けました。

$$
y = a x + b
$$

- `x` … 入力（家の広さ）
- `y` … 予測したい出力（値段）
- `a` … **傾き（かたむき）**。x が1増えたとき y がどれだけ増えるか
- `b` … **切片（せっぺん）**。x が0のときの y の位置（直線の高さ）

`a` と `b` を変えると、直線はいろんな向き・高さに動きます。急な坂にもなるし、ゆるい坂にもなる。上にも下にもずらせる。要するに、**`a` と `b` の2つの数字を決めれば、直線が1本ピシッと決まる**わけです。

この `a` と `b` のことを、機械学習では **パラメータ（parameter／モデルが持つ調整用のつまみ）** と呼びます。線形回帰の「学習」とは、ずばり **データに一番フィットする `a` と `b` を見つけること**。これに尽きます。難しい言葉に惑わされないでくださいね。やることは「つまみを2つ、いい感じに回す」だけです。

![傾きと切片を変えると直線が回転したり上下したりする様子の図](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-06/figure3.png)

## 予測と「誤差」──直線はどれだけズレてる？

ここで大事な疑問。「いい感じの直線」って、何をもって「いい」と言うんでしょう？

答えはシンプルで、**実際のデータの点に、なるべく近い直線がいい直線** です。逆に言うと、点から遠い直線はダメな直線。

そこで、各データ点について「本当の値」と「直線が言う予測値」の **差** を見ます。たとえば60㎡の家が本当は2800万円なのに、直線が「2600万円」と予測したら、200万円ズレていますよね。この **ズレ** のことを **誤差（ごさ／error）**、または **残差（ざんさ／residual、実測値と予測値の引き算）** と呼びます。

図でいうと、各点から直線へ **まっすぐ縦に降ろした線の長さ** が誤差です。この縦の隙間が全体的に小さいほど、直線はデータにフィットしている、というわけ。

![各データ点から直線へ縦に降ろした誤差(残差)を示した図](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-06/figure4.png)

## 最小二乗法：ズレを「二乗して」足す

じゃあ「全体のズレ」をどう測るか。ぜんぶの点の誤差を合計すればよさそうですが、ここにひとつ落とし穴があります。

誤差には **プラス（点が直線より上）とマイナス（点が直線より下）** があります。そのまま足すと、プラスとマイナスが打ち消し合って、本当はズレまくってるのに「合計ゼロ！」なんてことが起きちゃう。これじゃ困ります。

そこで登場するのが **二乗（じじょう／同じ数を掛け合わせること）** です。誤差を二乗すると、マイナスもプラスに変わります（マイナス×マイナスはプラスですからね）。この **二乗した誤差を全部足したもの** を小さくする直線を選ぶ。これが有名な **最小二乗法（さいしょうにじょうほう／Least Squares Method）** です。名前のとおり「二乗（したもの）を最小にする」方法、そのまんまですね。

二乗にはもう一つうれしい効果があって、**大きなズレをより強く罰する**んです。誤差2なら二乗して4ですが、誤差10なら二乗して100。10倍のズレが100倍のペナルティになる。だから最小二乗法は「とんでもなく外れてる点」を特に嫌がって、全体をバランスよくフィットさせてくれます。

図にすると、各誤差を1辺とする **正方形（＝二乗＝面積）** を描いて、その面積の合計がいちばん小さくなる直線を探すイメージです。

![各誤差を一辺とする正方形の面積の合計を最小にする最小二乗法のイメージ図](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-06/figure5.png)

## 「一番いい直線」はどうやって見つける？

二乗誤差の合計を、パラメータ `a`, `b` の関数として見たものを **損失関数（そんしつかんすう／loss function、モデルの間違い具合を測る関数）** と呼びます。「損（ズレ）がどれだけ大きいか」を表す関数、というイメージです。

この損失関数、`a` と `b` を変えると値が変わります。いい直線（`a`, `b`）なら損失は小さく、ダメな直線なら損失は大きい。これをグラフにすると、きれいな **お椀（わん）の形** になります。お椀のいちばん底が「損失が最小＝一番いい直線」の場所です。

じゃあ底をどう見つけるか。方法は2つあります。

1. **数式で一発で解く（最小二乗法の閉じた解）** … 線形回帰はシンプルなので、実は微分（第3回でやりましたね）を使うと、最適な `a`, `b` を計算式でズバッと求められます。
2. **少しずつ底へ転がり落ちる（勾配降下法）** … 第4回でやった **勾配降下法（こうばいこうかほう／gradient descent、坂を下って最小値を探す方法）** で、お椀の斜面をちょっとずつ下って底を目指す方法です。

方法1は線形回帰だから使える裏ワザで、方法2は **深層学習でも使う本命** です。今日は雰囲気だけ覚えておいてください。「お椀の底にボールを転がすように、パラメータを少しずつ調整していく」──これが学習の正体です。

![パラメータを変えると損失が変わるお椀型の谷を、ボールが底へ転がり落ちる勾配降下のイメージ図](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-06/figure6.png)

## コードで体感しよう（NumPy）

理屈はここまで。実際に動かしてみましょう。まずは **NumPy（ナムパイ／Pythonの数値計算ライブラリ）** だけで、最小二乗法の答えを直接計算してみます。

```python
import numpy as np

# データ: x=家の広さ(㎡), y=値段(万円)  ※わかりやすい架空データ
x = np.array([40, 60, 80, 100, 120], dtype=float)
y = np.array([2000, 2800, 3600, 4400, 5200], dtype=float)

# 最小二乗法の「閉じた解」で傾き a と切片 b を求める公式
#   a = (xとyがどれだけ一緒に動くか) / (xがどれだけ散らばっているか)
#   b = yの平均 - a * xの平均
x_mean = x.mean()
y_mean = y.mean()

# 分子: (x - xの平均) と (y - yの平均) を掛けて合計
numerator = np.sum((x - x_mean) * (y - y_mean))
# 分母: (x - xの平均) の二乗を合計
denominator = np.sum((x - x_mean) ** 2)

a = numerator / denominator   # 傾き
b = y_mean - a * x_mean       # 切片

print(f"傾き a = {a:.2f}, 切片 b = {b:.2f}")

# 求めた直線で「70㎡の家」の値段を予測してみる
new_x = 70
pred = a * new_x + b
print(f"70㎡の予測価格 = {pred:.1f} 万円")
```

実行すると、傾き `a = 40.00`、切片 `b = 400.00` あたりが求まり、70㎡の予測は約3200万円になります（このデータはきれいに直線に乗るよう作ったので、ぴったり `y = 40x + 400` になります）。**コンピュータがデータから直線を自分で見つけた**わけです。これが「学習」の第一歩！

上の公式が急に出てきて「なんだこれ」と思うかもしれませんが、これは損失関数（二乗誤差の合計）を `a` と `b` で微分してゼロと置いて解いた結果です。いまは「最小二乗法にはこういう便利な公式がある」くらいでOK。丸暗記する必要はありません。

## ライブラリを使えばもっと簡単

実務では、こういう計算は毎回手で書きません。**scikit-learn（サイキットラーン／Pythonの定番機械学習ライブラリ）** を使えば数行で済みます。

```python
import numpy as np
from sklearn.linear_model import LinearRegression

# scikit-learn は入力 x を「2次元」の形で欲しがるので reshape する
x = np.array([40, 60, 80, 100, 120], dtype=float).reshape(-1, 1)
y = np.array([2000, 2800, 3600, 4400, 5200], dtype=float)

model = LinearRegression()   # 線形回帰モデルを用意
model.fit(x, y)              # fit = データに合わせて学習(直線を見つける)

print(f"傾き = {model.coef_[0]:.2f}, 切片 = {model.intercept_:.2f}")

# 予測は predict を呼ぶだけ
print(model.predict([[70]]))  # 70㎡の予測価格
```

`fit()` が「学習」、`predict()` が「予測」。この **fit → predict** の流れは、線形回帰だけじゃなく、scikit-learn のほとんどのモデルで共通です。名前も「fit（フィットさせる）」「predict（予測する）」とそのまんまで、やさしいですよね。

![予測→誤差の計算→パラメータの調整→また予測、と回る学習ループの図](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-06/figure7.png)

## 「学習」って、結局こういうこと

ここまでを整理すると、線形回帰の学習はこんなループで説明できます。

1. とりあえず適当な直線（`a`, `b`）で **予測** してみる
2. 実際の値とくらべて **誤差（二乗誤差）** を測る
3. 誤差が小さくなる方向に `a`, `b` を **ちょっと調整** する
4. 1〜3をくり返して、お椀の底（損失最小）に近づける

このループ、実は **ニューラルネットワーク（神経回路をまねた仕組み）** でも、深層学習でも、まったく同じ骨組みなんです。違うのは「直線」が「もっと複雑な関数」に変わるだけ。つまり今日やった線形回帰は、深層学習という大きな建物の **土台の一番下の石** なんですね。ここが腹落ちしていると、この先ずっと楽になります。

![点群に引かれた直線が未知の点をぴたりと予測している完成イメージの図](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-06/figure8.png)

## まとめ

今日のポイントをおさらいします。

- **線形回帰** は、データの関係を1本の直線 `y = a x + b` で表して予測する、いちばんシンプルな機械学習です。
- 直線を決める `a`（傾き）と `b`（切片）が **パラメータ**。学習とは、この2つをデータに合うように調整することでした。
- 直線のよさは **誤差（残差）** で測ります。誤差を **二乗して合計** したものを最小にするのが **最小二乗法**。二乗するのは、プラスマイナスの打ち消しを防ぎ、大きなズレを強く罰するためでしたね。
- 二乗誤差の合計を **損失関数** として見ると、お椀の底が「一番いい直線」。そこへ向かうのが学習です。
- コードでは NumPy の公式でも、scikit-learn の `fit`／`predict` でも、あっさり直線を見つけられました。
