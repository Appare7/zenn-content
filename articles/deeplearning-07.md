この記事は「深層学習を一から」シリーズの第7回です。

前回（第6回）は **線形回帰（せんけいかいき／Linear Regression）** で、「点々の真ん中に、いい感じの直線を1本ひく」というのをやりましたよね。最小二乗法（さいしょうにじょうほう）という考え方で、直線を決めました。

でもあのとき、ちょっとモヤっとしませんでした？　「いい感じの直線を決める」って言うけど、**コンピュータはどうやって『いい感じ』かどうかを判断してるの？**　そして、どうやって**だんだん良くしていく**の？

今日はそのド真ん中、機械学習の心臓部にあたる **損失関数（そんしつかんすう／loss function）** と **勾配降下法（こうばいこうかほう／gradient descent）** の話です。この2つが分かると、「AIが学習する」という言葉の中身が、ぼんやりから急にクッキリ見えてきます。安心してください、たとえ話でゆっくり進めます。

![ロボットが谷を下りていくように、損失の一番低いところを目指す勾配降下法のイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-07/figure1.png)

## そもそも「学習する」ってどういうこと？

まず全体像から。機械学習でいう「学習」は、ざっくりこういうループです。

1. とりあえず適当な予測をしてみる
2. その予測が **どれくらい間違ってるか** を測る
3. 間違いが **少しでも小さくなる方向** に、パラメータ（predictの中の数字）をちょっと動かす
4. 2に戻る。これを何度も繰り返す

このうち「②間違いを測る」のが **損失関数**、「③少しでも小さくする方向に動かす」のが **勾配降下法** です。つまり今日の2つは、学習ループの**測る係**と**動かす係**なんですね。片方だけじゃ回りません。

## 損失関数（loss）とは：「間違いの大きさ」をはかるものさし

損失関数っていかつい名前ですが、やってることは超シンプル。**「予測が正解からどれだけズレてるか」を1つの数字にする**、それだけです。この数字を **損失（loss）** とか **誤差（error）** って呼びます。

ダーツを想像してください。的の中心（＝正解）を狙って矢を投げます。中心にドンピシャなら損失は0。外れれば外れるほど損失は大きくなる。この「中心からの距離」みたいなものが損失です。

![ダーツの的の中心と、そこからのズレを測るイメージ。予測と正解のギャップが損失](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-07/figure2.png)

大事なのは、**損失は小さいほど良い** ということ。テストの点数と逆で、0点に近づけたいやつです。だから学習のゴールは「損失をできるだけ小さくする」になります。

### いちばん定番の損失：平均二乗誤差（MSE）

回帰（数値を当てる問題）でよく使う損失が **平均二乗誤差（へいきんにじょうごさ／Mean Squared Error、略してMSE）** です。名前が長いだけで、中身は3ステップ。

1. 各データで「予測 − 正解」の**ズレ**を出す
2. そのズレを**二乗する**（マイナスを消して、大きいズレを強めに罰する）
3. 全部の平均をとる

数式で書くとこれだけです。難しく見えたら、上の日本語3ステップだけ覚えればOK。

$$
\text{MSE} = \frac{1}{n}\sum_{i=1}^{n}\left(\hat{y}_i - y_i\right)^2
$$

ここで $\hat{y}_i$（ワイハット）が予測、$y_i$ が正解、$n$ がデータの個数です。「予測と正解の差を二乗して、平均する」を記号にしただけ。

なぜ**二乗**するのか気になりますよね。理由は2つあって、①ズレのプラス・マイナスを打ち消さないため（3と−3が相殺すると困る）、②大きく外したときに損失をドカンと大きくして「そこは特にダメ！」と強調するため、です。

![点と直線の縦のズレを、小さな四角（二乗）として表した残差のイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-07/figure3.png)

Pythonで書くと、拍子抜けするくらい短いです。

```python
import numpy as np

# 正解ラベル（本当の値）
y_true = np.array([3.0, 5.0, 2.0, 8.0])
# モデルの予測値
y_pred = np.array([2.5, 5.5, 1.0, 7.0])

# ①ズレ → ②二乗 → ③平均、をそのまま1行で
mse = np.mean((y_pred - y_true) ** 2)

print(mse)  # 損失（小さいほど良い）
```

この `mse` の数字が小さくなるように、モデルの中の数字（パラメータ）を調整していく——それが学習の正体です。

:::message
損失関数は問題によって使い分けます。数値を当てる回帰ならMSE、犬か猫かのような分類（ぶんるい）なら **交差エントロピー（こうさエントロピー／cross entropy）** という別の損失をよく使います。分類の話は次回以降で出てきます。今日は「損失＝間違いのものさし」というイメージが持てれば十分です。
:::

## 損失を「小さくする」を地形でイメージする

さて、損失を測れるようになりました。次は「どうやって小さくするか」です。ここで一気にイメージしやすくなる、とっておきの見方を紹介します。それが **損失の地形（landscape）** です。

パラメータを横軸、そのときの損失を高さ（縦軸）だと思ってください。すると損失は、下の図みたいな **お椀（ボウル）型の谷** になります。谷の一番低いところが「損失が最小＝いちばん良いパラメータ」です。

![お椀型の谷の底に向かってボールが転がる、損失の地形（ランドスケープ）のイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-07/figure4.png)

こう見ると、学習のゴールがめちゃくちゃ直感的になります。**この谷の底までボールを転がして落とせばいい**んです。問題は「今どっちが下り坂か」を、地形全体を見渡せない状態でどう知るか。ここで勾配の出番です。

## 勾配（gradient）とは：坂の傾き、下りの向き

**勾配（gradient）** は、ひとことで言うと **「その場所の坂の傾き」** です。第2〜3回でやった微分（びぶん）の親戚で、「パラメータをちょっと動かしたら損失がどっち向きにどれくらい変わるか」を教えてくれます。

坂道に立って足元を触ると、「あ、こっちが下りだな」って分かりますよね。勾配はまさにそれ。**その地点で坂がいちばん急に上がる向き**を指します。だから、勾配と**逆向き**に進めば、いちばん急に**下れる**わけです。

![曲線に接する接線と、下り坂の向きを示す矢印。勾配は坂の傾きを表す](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-07/figure5.png)

ポイントを整理すると：

- 勾配が **急（大きい）** ＝まだ坂の途中、動かせば損失が大きく変わる
- 勾配が **ゆるい（0に近い）** ＝谷の底が近い、あまり変わらない
- 勾配が **ちょうど0** ＝平ら。谷の底に着いた合図

つまり勾配は「今どっちに、どれくらい下れるか」を教えてくれる**方位磁石**なんです。

## 勾配降下法（gradient descent）：目隠しで山を下りる

いよいよ主役、**勾配降下法** です。名前のとおり「勾配（坂）を下りていく方法」。やり方は驚くほどシンプルで、次を繰り返すだけです。

1. 今いる場所の **勾配（下りの向き）** を調べる
2. その向きに **少しだけ** 足を進める
3. 1に戻る

イメージは「濃い霧の中で、目隠しして山を下りる人」です。全体は見えないけど、足元の傾きだけは分かる。だから「今いちばん下ってる方向へ一歩」を、ひたすら繰り返す。それだけで、いつかは谷の底にたどり着けるんです。かしこいですよね。

![谷を少しずつ、点線の軌跡と足あとで下りていく勾配降下のステップ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-07/figure6.png)

これを式で書くと、こうなります（1個のパラメータ $w$ の場合）。

$$
w_{\text{新}} = w_{\text{今}} - \eta \times (\text{勾配})
$$

「今のパラメータから、勾配のぶんを **引く**」だけ。マイナスなのは、勾配は“上り”の向きを指すので、**逆に**進んで下りたいからです。ここで出てくる $\eta$（イータ）が超重要な **学習率（がくしゅうりつ／learning rate）** です。次の節でじっくりいきます。

### 実装イメージ：たった数行で「谷を下る」

言葉だけだとフワッとするので、実際に動くコードを見ましょう。ここでは損失を $f(w) = (w - 3)^2$ という単純なお椀型にします。答えは見てのとおり $w = 3$ で損失0ですが、**コンピュータはその答えを知らずに、勾配だけを頼りにたどり着けるか** を確かめます。

```python
# 損失関数：w=3 で最小になる、お椀型の関数
def loss(w):
    return (w - 3) ** 2

# 損失の勾配（微分）。(w-3)^2 を w で微分すると 2*(w-3)
def grad(w):
    return 2 * (w - 3)

w = 10.0        # スタート地点（とりあえず適当な場所）
lr = 0.1        # 学習率（1歩の大きさ）

for step in range(20):
    g = grad(w)          # ①今いる場所の勾配（下りの向き）を調べる
    w = w - lr * g       # ②勾配と逆向きに、少しだけ進む
    print(f"step {step:2d}: w={w:.3f}, loss={loss(w):.4f}")
```

これを実行すると、`w` が 10 → 8.4 → 7.12 …… と、じわじわ 3 に近づいて、損失がどんどん0へ落ちていきます。**答えを教えていないのに、坂を下るだけで正解にたどり着く**——これが勾配降下法の気持ちよさです。

線形回帰（第6回）も、中身はこれと同じ。「傾き」と「切片」という2つのパラメータについて勾配を計算して、損失（MSE）が小さくなる向きへ少しずつ動かしているだけなんです。パラメータが増えても、原理は1個のときとまったく同じ、というのが嬉しいところ。

## 学習率（learning rate）は「1歩の大きさ」

さっきの `lr = 0.1` が学習率でした。これは **1回でどれくらい大きく足を進めるか（歩幅）** を決める数字で、勾配降下法でいちばんチューニングに悩むやつです。大きすぎても小さすぎてもダメで、ちょうど良いところを探します。

![学習率が大きすぎて跳ねる・小さすぎて進まない・ちょうど良い、の3パターン比較](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-07/figure7.png)

- **大きすぎる**：歩幅がデカすぎて谷底を飛び越えてしまい、右へ左へ跳ね回って、いつまでも落ち着かない（最悪、どんどん外れて発散します）
- **小さすぎる**：安全だけど1歩がチマチマ。谷底までものすごく時間がかかる
- **ちょうど良い**：スイスイ谷底に近づく

最初は `0.1` や `0.01` あたりから試して、損失の下がり方を見ながら調整するのが定番です。「損失が下がらない／暴れる」ときは、まず学習率を疑う——これは覚えておくとかなり役立ちます。

## うまくいかないこともある：局所最適の谷

ここまでキレイなお椀型で話してきましたが、現実の損失の地形はもっとデコボコしていて、**谷が複数ある**ことがあります。深い谷（本当に一番いい場所＝**大域最適／global minimum**）のほかに、浅い谷（そこそこだけど最良じゃない場所＝**局所最適／local minimum**）があるんですね。

![浅い谷と深い谷があり、ボールが浅い谷にハマってしまう局所最適のイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/deeplearning-07/figure8.png)

勾配降下法は「今いる場所から下る」だけなので、運悪く浅い谷に入ると、そこから出られずに止まってしまうことがあります。「もっといい底があるのに、手前の谷で満足しちゃう」感じ。

「じゃあ実用にならないの？」と思うかもしれませんが、大丈夫です。実際の深層学習では、スタート地点をランダムにしたり、データを小分けにして勾配にちょっと“ゆらぎ”を与えたり（これを **確率的勾配降下法／SGD** と呼びます）といった工夫で、浅い谷にハマりにくくしています。この工夫の話は、また先の回でしっかり扱います。今日は「地形は必ずしもキレイなお椀じゃない」と頭の隅に置いておければ十分です。

## まとめ

今日は機械学習の心臓部、損失関数と勾配降下法を見てきました。

- **損失関数（loss）** は「予測が正解からどれだけズレてるか」を1つの数字にするものさし。回帰では **平均二乗誤差（MSE）** が定番。損失は **小さいほど良い**。
- 損失は **お椀型の谷** としてイメージできて、学習のゴールは **谷の底へボールを転がすこと**。
- **勾配（gradient）** は「その場所の坂の傾き」。下りの向きを教えてくれる方位磁石。
- **勾配降下法** は、勾配と逆向きに **少しずつ足を進める** ことを繰り返して谷を下る方法。式は `新しいw = 今のw − 学習率 × 勾配` だけ。
- **学習率** は歩幅。大きすぎると暴れ、小さすぎると進まない。
- 現実の地形はデコボコで、**局所最適** の浅い谷にハマることもある（工夫で対処する）。

「AIが学習する」の中身は、結局この**測って→下る**のループだったんですね。派手なニューラルネットワークも、根っこはこの繰り返しです。ここが腹落ちすると、この先ずっと景色が違って見えますよ。
