前回は「たくさんの数字をまとめて計算する道具」として線形代数（ベクトル・行列）を見ました。今回のテーマは **微分（びぶん）** と **連鎖律（れんさりつ／chain rule）** です。

「微分」と聞くと、高校の数学でつまずいた記憶がよみがえって、そっとブラウザを閉じたくなる人もいるかもしれません。でも大丈夫。深層学習で使う微分は、公式をゴリゴリ暗記する話じゃなくて、**「変化の割合を見るための道具」** というたった1つのイメージさえ掴めればOKなんです。しかもこの微分と連鎖律が、あとで出てくる **誤差逆伝播法（ごさぎゃくでんぱほう／backpropagation）** ——ニューラルネットワークが「学習」する仕組みそのもの——の心臓部になります。今回はそこまで、数式は最小限・イメージ最優先で一気に橋渡ししていきますね。気軽についてきてください。

![曲線に接する接線の傾きとして微分を表したイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/dl-02/figure1.png)

## そもそも、なぜ深層学習に微分が要るの？

深層学習では、モデルの中に **重み（おもみ／weight）** と呼ばれる大量の数字があります。学習とは、この重みを少しずつ調整して「予測のズレ（誤差）」をできるだけ小さくしていく作業です。

ここで問題になるのが、「重みをどっちに、どれだけ動かせば誤差が減るの？」という点。この“動かす向きと量”を教えてくれるのが微分なんです。微分は「この数字をちょっと増やしたら、結果はどっちにどれくらい動く？」を測る道具。だから微分がわからないと、モデルは自分をどう直せばいいのか分からない、というわけです。

まずはその「微分＝変化の割合」から、ゆっくりいきましょう。

## 微分＝変化の割合 ── 微分は何をしている？

**微分** をひとことで言うと、**「入力をほんの少し増やしたとき、出力がどれくらい変化するか（＝変化の割合）」** を求めることです。

自転車で坂道を走るところを想像してください。急な坂ほど「1メートル進むごとにグッと高さが上がる」ので、しんどいですよね。この“1メートル進むごとの高さの上がり具合”こそが **傾き（かたむき）** で、微分が求めているのはまさにこの傾きです。

![急な坂と緩やかな坂で傾きの大小を比べたイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/dl-02/figure2.png)

数式で書くと、関数 $f(x)$ の微分（傾き）はこう表せます。

$$
f'(x) = \lim_{h \to 0} \frac{f(x+h) - f(x)}{h}
$$

いかつく見えますが、中身は超シンプルです。$\dfrac{f(x+h)-f(x)}{h}$ は「$x$ を $h$ だけ増やしたときの、出力の増え方 ÷ 入力の増え方」、つまり **変化の割合そのもの**。そして $h \to 0$（エイチをゼロに近づける）は、「その増やす幅を限りなく小さくして、“今この瞬間”の傾きを見る」という意味です。

### なぜ「限りなく小さく」なの？

曲線を思いっきり拡大していくと、ほんの狭い範囲では**ほぼ直線**に見えます。地球が丸いのに、目の前の地面が平らに見えるのと同じですね。だから「ごく狭い範囲＝1点まわり」で見れば、曲がった関数も「傾きいくつの直線」として扱える。その1点での傾きを取り出すのが微分、というわけです。

![曲線を拡大するほど直線に近づく様子を示したイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/dl-02/figure3.png)

### 傾きの求め方を手で確かめる（NumPy）

いちばん有名な例、$f(x) = x^2$ でやってみます。この関数の微分は $f'(x) = 2x$ になることが知られています（公式）。$x=3$ なら傾きは $2 \times 3 = 6$ のはず。本当にそうなるか、Python の NumPy（ナムパイ／数値計算ライブラリ）で“ほんの少し動かして”確かめてみましょう。

```python
import numpy as np

# 傾きを測りたい関数
def f(x):
    return x ** 2  # x の2乗

x = 3.0        # 傾きを知りたい点
h = 1e-6       # ほんの少しだけ動かす幅（0.000001）

# 変化の割合 =（少し動かした後 − 少し動かす前）÷ 動かした幅
# 前後に動かして平均する「中心差分」だと精度が高い
slope = (f(x + h) - f(x - h)) / (2 * h)

print(slope)   # → 約 6.0（公式の 2x = 6 とほぼ一致！）
```

`h` を無限に0へ近づけることはコンピュータには無理なので、代わりに「めちゃくちゃ小さい数（$10^{-6}$）」を使って近似しています。結果はほぼ 6.0。公式で出した傾きと一致しましたね。「微分って、要は少しだけ動かして変化を見てるだけなんだ」と腹落ちすればこの節はクリアです。

## 合成関数ってなに？ ── 箱を順番に通すイメージ

連鎖律に進む前に、**合成関数（ごうせいかんすう）** を押さえます。名前は堅いですが、中身は「関数を数珠つなぎにしたもの」です。

たとえば「入力を3倍して1足す」関数 $g(x) = 3x + 1$ と、「2乗する」関数 $f(u) = u^2$ があるとします。この2つを順番に通すと、

$$
y = f(g(x)) = (3x + 1)^2
$$

になります。工場のベルトコンベアで、材料が1つ目の箱を通り、その結果が2つ目の箱を通って製品になる——そんなイメージです。深層学習のニューラルネットワークは、まさにこの「箱を何十個・何百個も直列につないだ、超巨大な合成関数」だと考えられます。

![入力が2つの箱を順に通って出力になる合成関数のイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/dl-02/figure4.png)

そうなると当然、「箱を積み重ねた関数全体の傾き（微分）って、どう求めるの？」という疑問が出てきます。そこで登場するのが、今回の主役 **連鎖律** です。

## 連鎖律（chain rule）＝ 傾きは掛け算でつながる

連鎖律をひとことで言うと、**「つなげた関数全体の傾きは、途中の傾きを掛け算すれば求まる」** というルールです。

歯車を思い浮かべてください。小さい歯車と大きい歯車が噛み合っていると、片方を回したとき、もう片方は歯数の比だけ速く（または遅く）回りますよね。それが何段も連なっていると、端っこの回転量は「各段の回転比を全部掛け算したもの」になります。連鎖律はまさにこれと同じ発想です。

![連なる歯車で回転が掛け算的に伝わる連鎖律のイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/dl-02/figure5.png)

さっきの合成関数 $y = (3x+1)^2$ で見てみましょう。途中の値を $u = 3x + 1$ と置くと、$y = u^2$ です。連鎖律はこう書けます。

$$
\frac{dy}{dx} = \frac{dy}{du} \times \frac{du}{dx}
$$

読み方は「$x$ が動くと $y$ はどれだけ動く？（$\frac{dy}{dx}$）は、$u$ が動くと $y$ が動く量（$\frac{dy}{du}$）と、$x$ が動くと $u$ が動く量（$\frac{du}{dx}$）を掛けたもの」。分数のように上下が“約分”されて $du$ が消えるイメージで覚えると気持ちいいです。

実際に計算すると、$\dfrac{dy}{du} = 2u$、$\dfrac{du}{dx} = 3$ なので、

$$
\frac{dy}{dx} = 2u \times 3 = 6u = 6(3x + 1)
$$

$x = 1$ なら $6 \times (3 \times 1 + 1) = 24$ です。念のため、展開して直接微分しても確かめられます。$y = (3x+1)^2 = 9x^2 + 6x + 1$ を微分すると $18x + 6$、$x=1$ で $24$。バッチリ一致しましたね。**バラバラの傾きを掛けるだけで、全体の傾きが出る**——これが連鎖律の威力です。

## 連鎖律が「誤差逆伝播法」の心臓になる話

ここからが今回のハイライト。ニューラルネットワークは「箱（層）を直列につないだ巨大な合成関数」でした。学習では「一番うしろで出た誤差を、各層の重みがどれくらい生んだか」を知りたい。つまり **「出力の誤差を、それぞれの重みで微分した値」** が欲しいわけです。

普通に考えると、層が100個あったら100回それぞれ微分し直すの…？とゾッとしますが、そこを救うのが連鎖律です。まず入力を前から後ろへ流して答えを出す（**順伝播／じゅんでんぱ**）。

![信号が前の層から後ろの層へ流れる順伝播のイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/dl-02/figure6.png)

そのあと、出力側の誤差から出発して、連鎖律の「傾きの掛け算」を**うしろから前へ**バケツリレーしていきます。各層では「1つうしろから受け取った傾き」に「その層自身の傾き」を掛けて、また1つ前へ渡すだけ。これを繰り返すと、全部の重みについての傾きが**一気に・効率よく**求まります。この後ろ向きの伝え方が **誤差逆伝播法（backpropagation）** です。

![誤差の勾配が後ろから前へ逆向きに伝わる逆伝播のイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/dl-02/figure7.png)

言葉だと難しく感じますが、やっていることは「連鎖律で傾きを掛け算しながら後ろから配っていくだけ」。難しい理論というより、**賢い順番の計算テクニック**なんだと思ってください。

### PyTorchなら微分は自動でやってくれる

うれしいことに、いまのフレームワークはこの連鎖律の計算を**全自動**でやってくれます。PyTorch（パイトーチ／深層学習ライブラリ）で、さっきの $y=(3x+1)^2$ の傾きを求めてみましょう。

```python
import torch

# requires_grad=True にすると「この値で微分してね」と印を付けられる
x = torch.tensor(1.0, requires_grad=True)

u = 3 * x + 1     # 1つ目の箱（合成関数の途中）
y = u ** 2        # 2つ目の箱

y.backward()      # ← ここで連鎖律を使って逆向きに微分を計算（逆伝播）

print(x.grad)     # → tensor(24.) 手計算した dy/dx = 24 と一致！
```

`y.backward()` の1行が、内部で連鎖律を回して「$x$ を動かすと $y$ がどれだけ動くか」を計算しています。手で出した 24 とピタリ一致。実務では自分で微分の式を書くことはほとんどなく、この仕組みに任せます。ただ「中身は連鎖律の掛け算なんだ」と知っているかどうかで、うまく学習しないときの理解度が段違いになります。

## 求めた傾きで、坂を下るように学習する

最後に「求めた傾きをどう使うの？」だけ触れておきます。傾き（勾配）は「誤差を増やす向き」を指しています。だったら、その**逆向き**に重みをちょっとずつ動かせば、誤差は減っていくはず。谷底を目指してボールが斜面を転がり下りるように、少しずつ低いほう（誤差が小さいほう）へ降りていく——このイメージが次回やる **勾配降下法（こうばいこうかほう）** です。

![勾配に沿って谷底へ一歩ずつ下る勾配降下のイメージ](https://raw.githubusercontent.com/Appare7/zenn-content/main/images/dl-02/figure8.png)

つまり今回の微分と連鎖律は、「坂の傾きを測る道具」。次回はその傾きを使って「実際に坂を下って学習する」ところへ進みます。

## まとめ

今回のポイントをおさらいします。

- **微分＝変化の割合（傾き）**。入力をほんの少し動かしたとき、出力がどれだけ動くかを測る道具。狭い範囲では曲線も直線に見える、という感覚が土台。
- **合成関数**は「箱を数珠つなぎにした関数」。ニューラルネットワークはこれの巨大版。
- **連鎖律（chain rule）**は「つなげた関数全体の傾きは、途中の傾きを掛け算すれば求まる」というルール。歯車のリレーをイメージ。
- その連鎖律を「出力の誤差から後ろ向きに配っていく」のが **誤差逆伝播法**。難しい理論というより、傾きを効率よく求める賢い計算順序。
- PyTorchの `backward()` はこれを全自動でやってくれる。でも中身が連鎖律だと知っていると理解が深まる。

微分アレルギーがあった人も、「要は変化の割合を掛け算でつないでるだけ」と思えれば、もう怖くないはずです。
